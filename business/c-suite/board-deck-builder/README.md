# board-deck-builder

## 用途

為高階主管建立或審查董事會簡報投影片，涵蓋敘事結構、財務報告、關鍵指標、策略更新及董事會適當的溝通標準。

## 觸發方式

當使用者要求建立董事會簡報、審查董事會投影片、構建董事會簡報、準備董事會材料、建立季度董事會更新或投資人更新簡報等相關問題時觸發。

範例觸發關鍵詞：董事會簡報、董事會投影片、董事會更新、季度董事會會議、董事會材料、投資人更新簡報。

## 使用步驟

1. **釐清會議目標** — 詢問：「這次會議結束後，你需要董事會決定什麼、感受什麼或相信什麼？」這決定簡報結構。
2. **找出敘事主軸** — 每份董事會簡報都在講一個故事。在建立投影片前，找出核心敘事弧：進展、挑戰、機會或拐點。
3. **構建簡報** — 套用標準董事會簡報架構（見下方），依會議類型客製化。
4. **起草內容** — 以誠實和自信的適當平衡建立每個章節。
5. **套用董事會溝通標準** — 先執行摘要、數字要有背景、決策以選項框架呈現、深度內容放附錄。
6. **審查董事會就緒度** — 檢查：請求是否清晰？艱難數字是否包含？壞消息是否被承擔而非埋藏？主要內容是否在 20 張投影片以內？

## 標準董事會簡報架構

### 季度更新簡報
1. **執行摘要**（1 張）— 主要成果、主要缺失、主要請求
2. **業務績效**（2–3 張）— 營收、成長、關鍵指標 vs. 計畫
3. **產品/客戶更新**（1–2 張）— 關鍵產品里程碑、重要客戶成交/流失
4. **團隊更新**（1 張）— 關鍵招聘、開放職位、值得注意的離職
5. **財務摘要**（2–3 張）— 損益摘要、現金狀況、跑道、燃燒趨勢
6. **關鍵決策/請求**（1–2 張）— 需要董事會意見或批准的具體事項
7. **前瞻展望**（1 張）— 下季優先事項和驅動計畫的關鍵假設
8. **附錄** — 詳細財務、客戶指標、深度產品數據

## 規則

### 必須
- 以執行摘要開頭——董事會時間緊迫
- 提前且清晰地呈現艱難真相（「我們未達到 Q3 目標，原因如下及我們的應對措施」）
- 以選項框架呈現決策，而非僅提供偏好的答案
- 主要簡報控制在 20 張投影片以內；深度內容放附錄
- 財務討論中始終包含現金狀況和跑道

### 禁止
- 將壞消息埋藏在腳注或附錄中
- 對重大決策僅呈現一個選項
- 不定義術語就使用行話
- 在任何財務討論中省略現金狀況和跑道
- 建立需要簡報者解釋投影片含義的簡報

## 範例

### 正確用法

**使用者：**「我們的 Q3 ARR 目標未達 15%。我下週有董事會會議，幫我建立這份更新。」

**回應：**
我來幫你構建這份簡報。缺失 15% 是顯著的，但如果你清晰地承擔責任並展示你理解原因，是可以處理的。

*執行摘要投影片：*
- Q3 ARR：$X.XM vs. $X.XM 目標（–15%）。主要驅動因素：[企業交易延遲/新客戶不足/客戶流失]
- 積極面：[一個真實的成果]
- 行動：[你正在採取的具體措施]
- 請求：董事會對[修訂的 Q4 計畫/管道審查流程]的意見

*根本原因分析投影片：*
「哪裡出了問題及為什麼」—— 2–3 個具體原因（不是「市場逆風」，而是具體的管道事項、轉換失敗、流失驅動因素）

*艱難真相：* 承擔缺失的領導者比解釋缺失的領導者更受董事會尊重。以「我們未達目標」而非「市場很艱難」開頭。

### 錯誤用法

**使用者：**「幫我建立董事會簡報。」

**回應：** 這裡是一個包含 10 張投影片的董事會簡報範本，涵蓋公司概覽、財務、產品路線圖、團隊和未來計畫⋯⋯

*（無會議背景、無具體請求、無情境特定指導的通用範本）*
