# change-management

## 用途

設計並執行組織變革管理策略，透過處理轉型的人員、結構與溝通面向，將組織抗拒轉化為採用動能，最大化採用率、最小化干擾並維持長期動力。

## 觸發方式

當使用者正在領導組織變革並需要結構化方法時觸發，例如：「我們正在推出新的營運模式」、「我們需要管理收購後的變革」、「我們的團隊對這個轉型有抵觸」、「為我們的系統遷移建立變革管理計畫」。

## 使用步驟

1. **定義變革** — 現狀（今日存在的）；未來狀態（變革後將存在的）；差距：人們必須停止、開始或以不同方式做的事；變革類型：結構性、流程性、技術性、文化性或策略性。
2. **評估變革準備度** — 評估 4 個因素：緊迫性（變革案例的說服力）、領導層對齊（高層是否可見地承諾？）、容量（組織在當前工作量下能否吸收此變革？）、歷史記錄（組織的變革成功紀錄如何？）。
3. **利害關係人分析** — 按「變革影響×當前態度（擁護者／中立／抗拒者）」繪製利害關係人地圖，識別關鍵贊助者、關鍵抗拒者與非正式影響者，為每個群體設計參與方式。
4. **建立變革敘事** — 為什麼現在（緊迫性）；改變了什麼（具體的未來狀態描述）；什麼保持不變（穩定因素以減少焦慮）；對你的意義（按利害關係人群體區分的 WIIFM）；提供什麼支援（資源、培訓、求助管道）。
5. **設計變革計畫** — 動員（領導層對齊、聯盟建立、早期擁護者）；溝通（多管道計畫）；賦能（培訓、工具、練習環境）；強化（採用認可、回饋循環）；維持（將變革嵌入流程與績效系統）。
6. **建立溝通計畫** — 訊息節奏（溝通什麼、何時、由誰、透過什麼管道）；主管工具包（談話要點、常見問題、升級路徑）；回饋機制。
7. **衡量變革採用** — 領先指標（認知、理解、採用意願）；落後指標（行為合規、績效指標）；回饋循環（脈衝調查、主管確認、使用資料）。

## 規則

### 必須

- 從變革準備度評估開始——低估容量時變革會失敗
- 為每個利害關係人群體建立差異化的 WIIFM 訊息
- 將主管培養為主要的變革傳達管道
- 設計強化系統——未被強化的變革會逆轉
- 以行為指標衡量採用，而非只是認知調查

### 禁止

- 將變革管理等同於溝通
- 在沒有可見且持續的領導層贊助的情況下啟動變革
- 低估中層管理人員的抗拒——他們能阻擋任何變革
- 將培訓等同於賦能——培訓是必要但不足以改變行為的條件

## 範例

### 正確用法

使用者：「我們正在從職能部門筒倉轉變為跨職能小組。250 人。60 天後開始。我擔心職能副總裁的抗拒。」

回應：評估準備度（中等——時程緊迫、VP 抗拒風險）、繪製 VP 抗拒者地圖附量身定製的參與方法、產出在保留職能專長的同時實現小組速度的變革敘事、設計主管串聯計畫，並建立附有 VP 對齊檢查點的 60 天動員日曆。

### 錯誤用法

使用者：「我們需要一個變革管理計畫。」

回應：「清楚溝通變革、培訓你的團隊、獲得領導層支持。」——沒有結構、沒有利害關係人分析、沒有採用衡量、沒有抗拒管理。
