# competitive-intel

## 用途

將原始競爭對手資料轉化為結構化的競爭情報，直接支持 C 級主管在定位、產品、定價與市場進入策略上的決策。

## 觸發方式

當使用者需要進行競爭分析、了解特定競爭對手，或釐清市場定位時觸發，例如：「分析我們的競爭格局」、「為競爭對手 X 製作 battlecard」、「為什麼我們一直輸給 X？」

## 使用步驟

1. **定義競爭框架** — 識別直接競爭對手（相同 ICP、類似解決方案）、間接競爭對手（不同方案、相同問題）與新興威脅（鄰近市場進入者）。
2. **建立競爭對手檔案** — 涵蓋公司概況、產品能力、定價模式、GTM 策略與近期動態。
3. **進行定位分析** — 在相關坐標軸上繪製競爭對手地圖，識別空白市場與爭奪領域，分析訊息模式。
4. **綜合勝負分析** — 若有勝負資料，識別主要原因，將敗因對應至競爭對手優勢，將勝因對應至可防禦優勢。
5. **提出策略含意** — 決定在何處積極競爭或讓步，識別需填補的產品缺口或訊息缺口。
6. **製作 battlecard（選用）** — 單頁格式，涵蓋對方話術、我方話術、差異化重點、可植入的質疑點、異議與回應。

## 規則

### 必須

- 明確區分已驗證的事實與推論訊號
- 標記資料來源與時效性
- 每項洞察都必須對應具體的業務決策
- 優先深入分析 2–3 個關鍵競爭對手，而非淺嚐多個
- 以銷售團隊能快速掃讀的語言撰寫 battlecard

### 禁止

- 將推測呈現為確認事實
- 在真空中進行分析——必須連結特定策略問題
- 忽略間接競爭對手或替代解決方案
- 在缺乏可觀察訊號的情況下主張定價資訊

## 範例

### 正確用法

使用者：「我們一直在企業級交易中輸給競爭對手 X。幫我了解原因並提出對策。」

回應：分析競爭對手 X 的企業級能力，根據所描述的交易模式識別造成失敗的 3 個產品缺口，製作附有異議處理的 battlecard，並建議 2 項短期訊息修正與 1 項納入產品路線圖的投資。

### 錯誤用法

使用者：「分析我們的競爭對手。」

回應：列出 10 家公司的維基百科級描述，沒有定位分析、沒有策略含意、沒有連結公司正在試圖做出的決策。
