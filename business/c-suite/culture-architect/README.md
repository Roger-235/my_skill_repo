# culture-architect

## 用途

透過行為框架、儀式設計與領導層對齊，設計、診斷並修復組織文化，將領導層的意圖轉化為可觀察的工作方式。

## 觸發方式

當使用者需要建立、修復或刻意轉變組織文化時觸發，例如：「我們的文化在擴展時正在崩解」、「幫我設計我們的文化」、「我們收購了一家公司，需要整合文化」、「我們的價值觀不真實」。

## 使用步驟

1. **診斷現有文化** — 識別有效的文化行為（強化）與失效或缺失的行為（介入），定位根本原因：激勵錯位、領導層建模缺口、規範模糊。
2. **定義目標文化** — 將願景性價值觀轉化為 3–5 條可觀察的行為規範，包含實踐樣貌、違反情形與領導層示範方式，識別 1–2 個「文化標誌性行為」。
3. **設計文化基礎建設** — 涵蓋招募訊號、入職儀式、認可制度、回饋規範與會議文化。
4. **對齊領導層行為** — 識別領導層必須改變或強化的 3 個具體行為，設計領導層問責循環。
5. **建立文化變革藍圖** — 快速勝利（30 天）、系統性變革（90 天）、嵌入組織設計（6–12 個月）。
6. **定義衡量指標** — 領先指標（具體行為追蹤）與落後指標（員工敬業度、留任率、eNPS）。

## 規則

### 必須

- 將每個文化願望錨定在具體、可觀察的行為上
- 在提出解決方案之前先診斷根本原因
- 明確處理領導層的榜樣行為——文化跟隨領導行為
- 設計回饋循環，讓文化能夠自我修正
- 考量較大型組織中的次文化差異

### 禁止

- 產出缺乏行為定義的價值觀清單
- 在脫離激勵與績效管理系統的情況下設計文化
- 將文化問題當作溝通問題處理（當根本是系統問題時）
- 忽略現有的文化優勢——應在有效之處上繼續建立

## 範例

### 正確用法

使用者：「我們有 200 人，曾讓我們在 30 人時成功的速度感與主人翁精神已經消失。發生了什麼？如何找回來？」

回應：診斷規模化時的文化漂移的結構性原因（管理層疊加、流程過重、風險趨避），識別 3 個需要找回的具體行為，設計一個彰顯主人翁精神的儀式，並產出附有領導層問責機制的 90 天藍圖。

### 錯誤用法

使用者：「幫我們建立更好的文化。」

回應：建議在牆上張貼價值觀、舉辦團隊外訪活動、發送每週文化通訊——沒有診斷、沒有行為具體性、沒有與制度的連結。
