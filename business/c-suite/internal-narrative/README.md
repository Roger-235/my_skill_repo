# internal-narrative

## 用途

撰寫能建立組織對齊、清晰說明策略決策、並在變革期間維護員工信任的內部溝通訊息。

## 觸發方式

當使用者需要向組織溝通策略決策、變革或方向時觸發，例如：「幫我撰寫關於我們轉型的全員大會訊息」、「起草裁員的內部溝通」、「我需要向團隊解釋這個策略轉向」、「撰寫組織重整的內部敘事」。

## 使用步驟

1. **定義溝通目標** — 員工必須理解什麼（認知目標）？希望他們感受到什麼（情感目標）？希望他們做什麼（行為目標）？
2. **評估受眾狀態** — 員工已知或相信什麼？他們會帶來哪些恐懼、問題或懷疑？他們對領導層的信任程度？
3. **建立敘事骨架** — 我們的過去：誠實框架先前狀態；發生了什麼：需要行動的訊號；我們的決定：清晰明確的決策聲明；為什麼這樣做：理由而非包裝；對你的意義：按受眾區分的具體影響；接下來是什麼：具體後續步驟與時間線。
4. **處理困難部分** — 承認不確定性、說出已做的取捨、提前處理最可能被問到的問題。
5. **起草溝通訊息** — 以格式匹配情境（備忘錄、全員大會腳本、Slack 訊息、電子郵件），以決策開頭而非鋪陳，使用白話文，保持合理簡短。
6. **檢視信任訊號** — 聽起來像真人寫的嗎？承認對人的影響了嗎？給了員工足夠資訊讓他們繼續工作嗎？

## 規則

### 必須

- 以決策開頭——員工不應讀到最後才知道發生了什麼變化
- 誠實承認人的影響，包括困難的事實
- 給員工清晰的「所以呢」——明天他們要做什麼不同的事？
- 為任何會引發問題的訊息提供主管談話要點
- 讓語調符合情境——裁員訊息不能聽起來像產品發布

### 禁止

- 使用委婉詞掩蓋責任（「精簡組織」、「策略性調整」而無解釋）
- 起草含有員工後來會發現不完整或誤導性資訊的訊息
- 以被動語態模糊誰做了決策及其原因
- 過度解釋過程而犧牲實質內容

## 範例

### 正確用法

使用者：「我們下週將裁減 15% 的員工。我需要撰寫全員大會公告和主管談話要點。」

回應：產出一份 400 字的直接全員大會備忘錄，以決策開頭，誠實解釋業務理由，承認人的代價，概述提供的支援，並包含 5 個附有坦誠建議回覆的主管常見問題。

### 錯誤用法

使用者：「撰寫關於我們裁員的全員大會訊息。」

回應：產出企業式用語（「困難但必要的策略演進」），不承認員工的感受，將決策埋在第三段，以空洞的「我們對未來感到興奮」結尾。
