# org-health-diagnostic

## 用途

跨結構、人才、文化與營運模式診斷組織健康狀況，找出功能失調的根本原因，並開具具有明確負責人與時程的針對性改善處方。

## 觸發方式

當使用者感受到組織功能失調、執行力不足或規模擴展摩擦，並希望診斷根本原因時觸發，例如：「我們的組織有問題但找不到癥結」、「我們執行力很差」、「各團隊一直沒有對齊」、「我覺得我們有人才問題但不確定在哪裡」。

## 使用步驟

1. **收集症狀資料** — 若未提供，請高層主管描述：過去 6 個月中前 3 個反覆出現的執行失誤、決策在哪裡卡住或被推翻、哪些團隊流失率最高或衝突最多、領導層花費過多時間管理的是什麼。
2. **從 5 個診斷維度進行評估** — 結構（管理幅度、匯報邏輯、功能重疊或問責缺口）；人才（能力缺口、角色人員錯配、關鍵人集中風險）；領導團隊（信任與衝突規範、決策品質、策略對齊）；文化與運作規範（資訊壟斷、心理安全、問責擴散）；營運模式（會議效率、目標設定品質、資源配置流程）。
3. **識別根本原因與症狀** — 將症狀對應至可能的根本原因，區分結構性、人才性與文化性原因，優先排序：修復哪些根本原因能解決最多症狀。
4. **產出診斷報告** — 每個維度的健康評分（健康／風險／危急）、前 3 大根本原因（附佐證）、優先改善清單。
5. **設計介入措施** — 對每個優先根本原因推薦介入措施（結構性、人才性、文化性或流程性），指定負責人、時程與追蹤的領先指標。

## 規則

### 必須

- 明確區分症狀與根本原因
- 即使部分維度健康，也要評估全部 5 個維度
- 優先排序介入措施——無法同時修復所有問題
- 為每個介入措施指定負責人
- 將領導團隊納入診斷對象，而非只看其下方的組織

### 禁止

- 純粹從高層主管的框架診斷——應主動尋找反面證據
- 將重組作為每個問題的預設解決方案
- 產出沒有優先順序的 20 個問題清單——這對任何人都沒有幫助
- 忽略領導層行為可能是首要根本原因的可能性

## 範例

### 正確用法

使用者：「我們一直錯過季度目標，每次原因都不同。180 人，18 個月內從 40 人成長到現在。」

回應：診斷快速成長的規模化摩擦，識別 3 個根本原因（目標設定流程崩潰、管理層尚未為規模培養、跨職能問責缺口），為 5 個維度評分，並產出附有具體負責人和指標的 90 天介入藍圖。

### 錯誤用法

使用者：「我們的組織不健康。」

回應：「考慮改善溝通、釐清角色職責、建立更強大的文化。」——沒有診斷流程、沒有根本原因分析、沒有優先順序。
