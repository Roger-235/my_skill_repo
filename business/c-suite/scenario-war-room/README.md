# scenario-war-room

## 用途

執行嚴謹的情境規劃與策略兵棋推演，在不確定性下對決策進行壓力測試，並建立附有應變行動手冊的抗韌性決策框架。

## 觸發方式

當使用者希望對策略進行壓力測試、為不確定性做準備或推演競爭或市場情境時觸發，例如：「推演我們的產品發布」、「幫我們進行衰退情境規劃」、「如果最大競爭對手降價 40% 會怎樣？」、「對我們的 3 年計畫進行壓力測試」。

## 使用步驟

1. **定義策略問題** — 明確表述被測試的核心決策或假設。
2. **識別關鍵不確定性** — 列出 8–12 個可能顯著影響結果的因素，按主題分類：市場需求、競爭動態、監管、技術、總體經濟。選出 2 個高度不確定且高度影響的軸線作為情境矩陣軸。
3. **建立情境矩陣** — 以 2 個軸線建立 4 個象限情境，為每個情境取一個生動的名稱，以 2–3 段描述每個世界（市場狀況、競爭格局、客戶行為、總體經濟背景）。
4. **對每個情境壓力測試現行策略** — 什麼有效？什麼會崩潰？預估的財務／策略影響？出現哪些早期預警訊號意味著此情境正在成真？
5. **識別穩健行動** — 在多個情境下均有效的「無悔行動」、情境特定的「重大賭注」、在多個情境下應避免的「策略殺手」，以及保留彈性的對沖選項。
6. **建立應變行動手冊** — 每個情境的觸發條件（具體可觀察訊號）、30/60/90 天回應計畫、預授權決策。
7. **綜合整理決策建議** — 無論情境如何現在都應採取的行動、應分階段並設審查觸發點的決策、領導層應追蹤的監控儀表板。

## 規則

### 必須

- 將情境錨定在合理而非虛構的未來
- 至少包含一個挑戰高層主管最偏愛假設的情境
- 區分無悔行動與情境依賴性賭注
- 建立可執行的應變行動手冊，而非只是情境敘事
- 為每個情境定義具體的早期預警訊號

### 禁止

- 建立情境只是為了驗證預先確定的結論
- 只使用最佳情境和最壞情境（這遺漏了大部分情境空間）
- 產出無法回饋至實際決策的分析
- 讓情境抽象到無法操作化的程度

## 範例

### 正確用法

使用者：「我們計劃明年融資 5000 萬美元並將人力加倍。推演這個計畫可能出錯的情境。」

回應：識別 2 個軸線（資本市場狀況×市場需求），建立 4 個情境包含「資本荒」與「需求疲軟」的情況，對每個情境壓力測試招聘計畫，識別 3 個無悔行動，並建立與領先收入指標掛鉤的招聘速度進行／不進行觸發框架。

### 錯誤用法

使用者：「我們的業務有哪些不同情境？」

回應：「最佳情境：一切順利。基準情境：適度成長。最壞情境：衰退。」——沒有軸線、沒有壓力測試、沒有應變行動、沒有早期預警訊號。
