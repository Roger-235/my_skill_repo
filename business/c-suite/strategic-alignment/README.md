# strategic-alignment

## 用途

識別並解決跨領導團隊、職能部門與組織優先事項的策略錯位，恢復一致且高速度的執行力。

## 觸發方式

當使用者識別到阻礙執行的錯位問題時觸發，例如：「我們的領導團隊沒有對齊」、「業務和產品一直在衝突」、「每個人的優先事項都不同」、「我們同意了策略但什麼都沒發生」、「幫我們對齊 OKR」。

## 使用步驟

1. **診斷錯位類型** — 策略性錯位（對目標、方向或優先事項有分歧）、營運性錯位（同意做什麼，但不同意如何做或由誰負責）、資源性錯位（目標存在但資源配置不支持目標）、資訊性錯位（不同團隊依據不同的事實運作）。
2. **繪製錯位地圖** — 哪些方在錯位（領導團隊、跨職能團隊、組織層級）？在哪些具體問題或決策上？何時開始錯位？代價是什麼（延誤、重工、流失、客戶影響）？
3. **識別根本原因** — 缺乏書面的共同策略、目標在頂部設定但未有效向下傳達、跨職能激勵競爭、未解決的既有分歧、阻礙坦誠討論的領導團隊動態、優先事項過多（缺乏強制選擇機制）。
4. **設計對齊介入措施** — 選擇適當機制：對齊工作坊、優先事項釐清（強制排序前 3 大優先事項）、RACI 重建、OKR 調和、領導團隊重置。
5. **產出對齊成果物** — 附強制優先順序排名的一頁策略摘要、跨職能相依性地圖、最具爭議決策的決策權限矩陣。
6. **設計對齊維護系統** — 每月領導層對齊確認（30 分鐘議程）、每季優先事項審查（含明確取捨討論）、跨職能衝突的升級協議。

## 規則

### 必須

- 在提出解決方案之前診斷錯位類型
- 處理根本原因，而非只是表面症狀
- 產出書面對齊成果物——口頭對齊會消退
- 設計維護系統——對齊不是一次性事件
- 具體說明需要解決的決策或優先事項

### 禁止

- 當問題是結構性或激勵性問題時，將錯位視為溝通問題
- 設計迴避領導層需要做出艱難取捨的對齊流程
- 讓工作坊取代清楚的書面決策
- 以更多 OKR 作為因 OKR 過多而造成的錯位問題的解決方案

## 範例

### 正確用法

使用者：「我的領導團隊在季度業務審查中同意了策略，但之後每個職能部門各行其是。我們已進入本財年第 6 個月，每個團隊都在為不同的事情優化。」

回應：診斷為資源性＋營運性錯位，識別缺失的強制優先機制，設計一個附有前置作業的 3 小時領導層對齊會議，產出一份附有明確取捨的一頁優先順序清單，並建立附有 4 個常設議題的每月對齊儀式。

### 錯誤用法

使用者：「幫我們對齊。」

回應：「安排外訪活動、建立共同 OKR、改善跨職能溝通。」——沒有診斷、沒有根本原因、沒有成果物、沒有維護系統。
