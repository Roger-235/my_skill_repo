# qms-audit-expert

## 用途

依據 ISO 13485:2016 和 FDA 21 CFR Part 820，對醫療器材製造商進行品質管理系統稽核，應用 ISO 19011 稽核原則提供稽核規劃、執行、發現分類、不符合事項報告及矯正措施跟進。涵蓋供應商稽核、流程稽核、系統稽核、認證稽核及法規查廠準備。

## 觸發方式

當使用者需要以下協助時啟用：
- 規劃和執行 ISO 13485:2016 或 FDA 21 CFR 820 內部 QMS 稽核
- 執行或準備供應商資格認定和監督稽核
- 準備第三方認證機構稽核（第一/第二階段/監督）
- 準備 FDA 品質系統查驗（QSIT 方法）
- 為特定 QMS 流程或條款開發稽核查核表
- 分類稽核發現和起草不符合事項報告

使用者指令範例：「QMS 稽核」、「ISO 13485 稽核」、「內部稽核」、「供應商稽核」、「FDA 查廠」、「QSIT」、「稽核查核表」、「稽核發現」、「NCR」、「查廠準備」。

## 使用步驟

### 第一步：稽核計畫規劃
- 定義年度稽核計畫目標：評估 QMS 符合性、有效性和合規性
- 安排稽核確保在 12 個月內涵蓋所有 ISO 13485 條款和關鍵流程
- 風險性排程：較高風險流程（設計管制、CAPA、生產）更頻繁稽核
- 指派合格的獨立稽核員（建議取得主任稽核員認證：IRCA、ASQ CQA）

### 第二步：設計管制稽核重點（ISO 13485 §7.3 / FDA 820.30）
- 每台器材是否維護設計歷史檔案（DHF）？
- 設計輸入是否有文件並已核准（使用者需求、法規要求、風險管理輸出）？
- 設計輸出是否可追溯至輸入？
- 是否在計劃階段進行了設計審查並有適當人員？
- 設計驗證（設計是否符合輸入？）是否有已核准的方案和報告？
- 設計確認（器材是否在代表性條件下符合使用者需求？）是否有文件？
- 設計移轉是否有文件？
- 設計變更是否受控並評估對現有核准的影響？

### 第三步：FDA QSIT 稽核方法
FDA 查廠員使用品質系統查驗技術（QSIT）方法，包含四個強制性子系統：

**管理管制**：管理責任（820.20）；品質稽核（820.22）；人員（820.25）

**設計管制**（820.30）：稽核 DHF、設計輸入/輸出記錄、驗證/確認方案和報告、設計移轉文件、設計變更管制

**CAPA**（820.100）：稽核 CAPA 程序、開放和已關閉的 CAPA、根本原因分析品質、有效性驗證記錄

**生產與製程管制**（820.70）：稽核製程確認主計畫、IQ/OQ/PQ 記錄、設備資格/校驗記錄、環境監控、批次記錄

### 第四步：發現分類

**嚴重發現（FDA）/ 重大不符合事項（ISO）**：
- QMS 要素或管制措施的系統性失敗
- 病患安全風險或觸發法規通報義務
- 缺少必要程序或管制措施
- 先前稽核週期的重複發現

**重大不符合事項**：顯著未符合條款要求；若未解決則認證面臨風險

**輕微不符合事項**：孤立失誤；條款部分實施

**觀察事項/OFI**：尚未構成不符合事項；呈現趨勢的關注點

不符合事項陳述：「客觀證據 [描述證據] 不符合 [ISO 13485:2016 §X.X / 21 CFR 820.XX]，該條款要求 [具體要求文字]。」

## 規則

### 必須
- 每項發現都基於具體的、有文件的客觀證據 — 不是印象或傳聞
- 在每項不符合事項中引用確切的 ISO 13485:2016 條款或 21 CFR 820 章節編號
- 對稽核排程應用風險性方法（較高風險流程更頻繁）
- 確保稽核員獨立性：稽核員不得稽核其實施或管理的流程
- 驗證矯正措施的根本原因分析 — 不僅是行動完成情況
- 在發出書面報告前進行呈現所有發現的結束會議

### 禁止
- 在沒有支持客觀證據的情況下發出發現
- 允許受審者在未提供額外證據的情況下談判降低發現分類
- 在沒有基於證據的有效性驗證的情況下關閉重大 NC
- 在未提前與受審者分享稽核計畫的情況下進行稽核
- 將稽核發現視為法律指控 — 發現是基於證據的改善機會

## 範例

### 正確用法
使用者：「為我們的第 IIb 類輸液泵執行設計管制稽核。」

回應：為 21 CFR 820.30 和 ISO 13485:2016 §7.3 生成針對性查核表；要求 DHF 索引以驗證完整性；抽樣設計輸入以追溯至使用者需求和法規要求；審查 IEC 60601-1、IEC 60601-1-2（EMC）、ISO 10993 生物相容性的設計驗證測試方案，以及 IEC 62304 軟體確認；查核臨床使用模擬的設計確認方案/報告；驗證設計移轉文件與製造程序的連結；審查設計變更記錄的影響評估。

### 錯誤用法
使用者：「公司有設計管制程序文件。他們是否合規？」

回應：「是的，有程序意味著他們符合設計管制。」— 此為無效做法：程序的存在不能證明合規；稽核員必須通過審查 DHF 記錄、設計審查會議記錄、驗證/確認報告以及實際器材的設計移轉文件來驗證程序是否已實施。
