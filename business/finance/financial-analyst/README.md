# financial-analyst

## 用途

執行嚴謹的財務分析，將原始財務數據轉化為可行的業務洞察，支援投資決策、營運評估與策略規劃，涵蓋財務建模、比率分析、估值與預測。

## 觸發方式

當使用者需要以下協助時觸發：
- 財務建模（P&L 預測、現金流預測）
- 比率分析（獲利能力、流動性、槓桿）
- 企業估值（DCF、EV/EBITDA、本益比）
- 預算對比實際差異分析
- 投資報酬率（ROI）計算與評估

## 使用步驟

1. 確認分析目標（此分析需支援的決策）
2. 評估提供的財務數據，標記缺失項目
3. 選擇適當的分析框架執行核心分析：
   - 獲利能力：毛利率、EBITDA 利潤率、淨利率、ROIC
   - 流動性：流動比率、速動比率、現金轉換週期
   - 槓桿：負債/股東權益、利息覆蓋率、淨負債/EBITDA
   - 估值：DCF、EV/EBITDA、可比公司分析
4. 如需要，建立財務模型（收入拆解、成本結構、P&L 預測、現金流）
5. 執行情境分析（基本案例、樂觀案例、悲觀案例）
6. 識別關鍵驅動因素（最影響結果的 3–5 個變數）
7. 詮釋結果（將數字轉化為業務啟示）
8. 提出決策導向的建議

## 規則

### 必須
- 明確標記所有假設及其來源（管理層指引、行業基準、歷史趨勢）
- 提供情境分析——絕不只提供單點估算
- 說明使用的估值方法及其局限性
- 標記資料品質問題或缺失資訊

### 禁止
- 提供 DCF 估值時不揭露折現率和終端成長率假設
- 在未與行業可比值對標的情況下外推成長率
- 混淆 EBITDA 與現金流——必須清楚區分
- 在現金流分析中遺漏營運資金變動

## 範例

### 正確用法
```
收入預測（基本案例）：
- FY2024A：$12.4M | FY2025E：$17.3M（+40% 年增率）
- 關鍵假設：每季新增 35 個客戶，平均合約價值 $120K（基於 H2 2024 管線覆蓋率 2.3x）
- 敏感性：客戶成長率 ±10% → 收入影響 ±$1.7M
- 悲觀情境：流失率上升至 15% → FY2025E 收入 $15.1M

EBITDA 利潤率：18%（vs. SaaS 可比中位數 22%）——差距來自 S&M 佔收入 35% 偏高；預計隨 CAC 回收期達到 14 個月後於 FY2026 正常化至 28%。
```

### 錯誤用法
```
明年收入將成長 40%。利潤看起來不錯。公司大約值 5000 萬美元。建議投資。
```
