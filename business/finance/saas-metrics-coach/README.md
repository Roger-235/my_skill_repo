# saas-metrics-coach

## 用途

指導 SaaS 創辦人與營運者正確計算、詮釋並改善核心 SaaS 指標，包括 ARR、MRR、流失率、NRR、CAC、LTV、回收期與 40 法則，並與行業基準進行比較。

## 觸發方式

當使用者需要以下協助時觸發：
- 計算或理解 SaaS 核心指標（ARR、MRR、NRR、CAC、LTV）
- 診斷 SaaS 業務健康問題
- 與行業基準對標
- 改善單位經濟效益（Unit Economics）
- 準備投資人指標報告

## 使用步驟

1. 審計指標基礎（確認使用者是否正確計算核心指標）
2. 計算核心 SaaS 指標：
   - MRR / ARR（月度/年度經常性收入）
   - MRR 組成：新增 + 擴張 − 縮減 − 流失
   - 客戶流失率 vs. 收入流失率
   - 淨收入留存率（NRR）
   - 毛收入留存率（GRR）
   - 客戶獲取成本（CAC）
   - 客戶終身價值（LTV）= （ARPU × 毛利率）/ 收入流失率
   - LTV:CAC 比率（基準：≥3x）
   - CAC 回收期
   - 40 法則（收入成長率 % + EBITDA 利潤率 %）
3. 與階段適當的行業規範對標
4. 識別最薄弱的指標並診斷根本原因
5. 推薦改善槓桿（針對每個表現不佳指標的具體行動）
6. 建立改善影響模型

## 規則

### 必須
- 區分收入流失率與客戶流失率——兩者傳達不同的業務訊息
- 正確計算 NRR——這是最重要的 SaaS 健康指標
- 與階段和市場區隔適當的同行對標（SMB SaaS 與企業 SaaS 標準不同）
- 在 LTV 計算中使用毛利率，而非收入

### 禁止
- 混淆預訂金額（Bookings）與收入（ARR）——預訂是承諾，ARR 是已確認收入
- 在未驗證擴張收入是否正確記錄的情況下，報告 NRR 超過 100%
- 將總人力成本計入 CAC——只計入銷售和行銷費用
- 當總體 NRR 看起來健康時，忽略同期群層級的留存率

## 範例

### 正確用法
```
NRR 計算：
期初 MRR：$500K
+ 擴張：$45K（追加銷售 + 座位增加）
− 縮減：$12K（降級）
− 流失 MRR：$18K
= 來自現有客戶的期末 MRR：$515K
NRR = $515K / $500K = 103%

基準：SMB SaaS 表現強勁（中位數 ~100%）；中型市場優秀（中位數 ~105%）。
建議：擴張動作有效——優先推行主動 QBR 計畫，透過減少縮減將 NRR 推至 108%+ 。
```

### 錯誤用法
```
本月新增了很多客戶所以 NRR 表現很好。ARR 成長了 20%，所以我們做得很好。
```
