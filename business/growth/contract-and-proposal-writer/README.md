# contract-and-proposal-writer

## 用途

起草商業提案、RFP 回應、工作說明書（SOW）、服務協議及合約語言，適用於銷售和業務發展情境，產出能贏得交易、保護雙方並設定清晰交付期望的文件。

## 觸發方式

當使用者需要以下協助時觸發：
- 撰寫商業提案或業務開發提案
- 回應 RFP（詢價書）的技術或商業部分
- 起草工作說明書（SOW）
- 建立服務協議或商業合約語言
- 撰寫 NDA 或保密協議

## 使用步驟

1. 確認文件類型與目的（提案：說服；SOW：定義範疇；協議：管理關係）
2. 了解客戶背景（問題、評估標準、決策流程）
3. 起草執行摘要（問題 → 解決方案 → 關鍵效益 → 商業摘要 → 行動呼籲）
4. 撰寫工作範疇/解決方案章節（清楚定義包含/排除項、交付物、時程、驗收標準）
5. 呈現商業條款（定價表、付款時程、費用政策、變更訂單流程）
6. 定義角色與責任（客戶 vs. 供應商的 RACI；關於客戶投入的假設）
7. 處理條款與條件（智慧財產權、保密、責任、終止、適用法律）
8. 加入信用憑證（相關案例研究、團隊簡介、認證、客戶標誌）
9. 撰寫結尾（後續步驟、簽名欄、有效期限）
10. 審查法律風險（標記需要法律審查的條款）

## 規則

### 必須
- 清楚定義範疇以外的內容——模糊的範疇是合約糾紛的主因
- 所有 SOW 包含變更訂單/變更流程
- 明確說明付款條款（淨 30 天、里程碑制等）
- 標記需要法律審查的條款——不要將法律語言呈現為最終版本而不加說明

### 禁止
- 使用模糊的範疇語言（「按需提供的諮詢服務」）
- 省略交付物的驗收標準定義
- 在未標記供法律審查的情況下包含無限責任條款
- 在正式提案中不附細項明細表呈現定價

## 範例

### 正確用法
```
範疇：Salesforce CRM 導入——第一階段

範疇內：
- 最多 5 個自訂物件的配置
- 從 HubSpot 遷移最多 50,000 筆聯絡人記錄
- 2 天最終使用者培訓（最多 20 位參與者）
- 上線支援（5 個工作日）

範疇外：
- 客製化 Apex 開發
- 與 ERP 系統的整合（第二階段）
- 上線後的持續管理服務

變更訂單流程：任何範疇變更需書面核准；超過 $5,000 的變更在開始工作前需要簽署變更訂單。
```

### 錯誤用法
```
我們將為您的團隊導入 Salesforce，並做任何需要做的事情讓它運作。價格：$50,000。告訴我們何時開始。
```
