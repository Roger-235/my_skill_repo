# customer-success-manager

## 用途

擔任客戶成功經理（CSM），設計上手計畫、建立 QBR 報告架構、構建客戶健康評分框架、制定擴張攻略，並管理高風險客戶干預，驅動客戶留存與擴張。

## 觸發方式

當使用者需要以下協助時觸發：
- 設計客戶上手計畫（Onboarding Plan）
- 準備季度業務回顧（QBR）
- 建立客戶健康評分模型
- 建立擴張銷售攻略
- 管理高流失風險客戶
- 制定客戶成功策略

## 使用步驟

1. 定義客戶目標（記錄客戶期望的業務成果，而非產品功能）
2. 設計上手計畫（以里程碑為基礎：啟動會議 → 技術設定 → 首次價值 → 採用 → 擴張準備）
3. 建立健康評分模型（產品採用 40%、業務成果達成 30%、關係強度 20%、支援健康 10%，評分 0–100）
4. 建立 QBR 架構（議程：目標回顧 → 使用數據亮點 → 達成成果 → 挑戰與解決 → 路線圖對齊 → 下期目標 → 續約/擴張討論）
5. 制定擴張攻略（識別擴張觸發點；定義外展序列和報價）
6. 高風險干預計畫（識別風險訊號；分配風險層級；起草挽救計畫）
7. 計算客戶 ROI（量化交付的業務價值 vs. 合約價值）
8. 續約管理（續約前 120 天清單）

## 規則

### 必須
- 始終將成功錨定於客戶的業務成果，而非單純的產品使用量
- 包含與客戶共同簽署的互利成功計畫（MSP）
- 每週對健康評分計分並追蹤趨勢——單點評分具有誤導性
- 在每次續約對話前量化 ROI

### 禁止
- 進行只是產品演示或功能更新的 QBR
- 僅根據登入頻率定義健康評分——這是滯後且膚淺的訊號
- 僅在續約時才接觸高風險客戶——在第一個風險訊號出現時就介入
- 在擴張對話中承諾路線圖功能而沒有獲得產品團隊的對齊

## 範例

### 正確用法
```
健康評分：Acme Corp——評分：67/100（橘色，從 74 下降）
產品採用：58/100——購買的 10 個座位中只有 3 個活躍；核心工作流程功能只有 1 個使用者在使用
業務成果：70/100——達成 3 個既定目標中的 2 個；報告自動化仍未上線
關係強度：80/100——強力擁護者（VP Finance）；主管贊助商確認參加 QBR
支援健康：55/100——2 個 P2 工單超過 14 天未解決

風險訊號：座位使用率下降。根本原因：IT 配置延遲。干預：指派專屬上手會議；於 2026-03-25 前納入 IT 利害關係人。
```

### 錯誤用法
```
客戶健康：他們上週登入了所以看起來沒問題。QBR 議程：給他們看新功能並問他們是否想要續約。
```
