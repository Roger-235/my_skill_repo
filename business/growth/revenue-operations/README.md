# revenue-operations

## 用途

設計並優化營收運營（RevOps）框架，涵蓋銷售漏斗分析、CRM 數據品質、潛在客戶評分、管線預測、領土規劃，以及銷售、行銷與客戶成功團隊之間的 GTM 對齊，建立統一的營收引擎。

## 觸發方式

當使用者需要以下協助時觸發：
- 設計或優化營收運營框架
- 分析管線健康狀況
- 建立潛在客戶評分模型
- 改善 CRM 數據品質
- 建立銷售預測
- 規劃銷售領土與配額

## 使用步驟

1. 繪製營收漏斗（定義從頂部漏斗到成交到擴張的各個階段；建立進出階段標準）
2. 審計漏斗轉換率（測量每個階段的轉換；識別最大的流失點）
3. 建立潛在客戶評分模型（人口統計適配分 + 行為分 → 總分 → MQL 閾值）
4. 設計 CRM 數據模型（標準欄位、每階段必填欄位、數據所有權、品質規則）
5. 建立銷售預測（管線 × 階段加權機率 = 加權預測；加入業代層級預測；承諾/上行/管線類別）
6. 領土與配額規劃（定義領土標準、配額方法論、新員工斜坡時程）
7. GTM 對齊流程（定義行銷→銷售→CS 之間的 SLA；設定共同指標）
8. 營收指標儀表板（ARR 預訂、管線覆蓋率、成交率、ACV、銷售週期長度、CAC）
9. 識別營收漏損（審計：停滯 >30 天的交易、未記錄後續步驟、折扣 >20% 未核准）

## 規則

### 必須
- 精確定義階段退出標準——模糊的階段導致預測不準確
- CRM 衛生不可妥協：每個階段必須設定成交日期、後續步驟和金額
- 在報告中區分管線覆蓋率（數量）與預測（品質）
- 對齊單一的營收數據真實來源——碎片化的報告會摧毀信任

### 禁止
- 使用活動指標（撥打電話數、發送郵件數）作為管線健康的代理指標
- 在未用歷史成交數據驗證的情況下建立潛在客戶評分模型
- 在沒有由下而上產能模型的情況下設定配額——由上而下的配額會打擊業代積極性
- 在沒有成交率背景的情況下報告管線覆蓋率

## 範例

### 正確用法
```
漏斗轉換率審計：
MQL → SQL：28%（基準 30%）——接近目標；透過更精準的 ICP 定位改善
SQL → 商機：62%（基準 55%）——強勁的探索資格審查
商機 → 提案：71%
提案 → 成交：22%（基準 35%）——紅色警報。主要問題：提案階段的競爭失敗。建議進行成敗分析並更新競爭戰情手冊。

管線覆蓋率：3.2x（目標 ≥3x）✓
預測：$1.8M 承諾 + $420K 上行 vs. $2.1M 目標——缺口 $300K。風險：2 個交易在法務審查超過 21 天。
```

### 錯誤用法
```
銷售進展良好。管線看起來不錯。我們有很多潛在客戶，團隊在打電話。營收應該沒問題。
```
