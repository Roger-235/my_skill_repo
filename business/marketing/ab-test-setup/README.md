# ab-test-setup

## 用途

設計具統計嚴謹性的 A/B 測試，涵蓋假設陳述、樣本量計算、成功指標定義與分析計畫，確保產出可信的測試結果。

## 觸發方式

當使用者詢問「設定 A/B 測試」、「設計實驗」、「測試兩個版本」、「執行分割測試」，或需要結構化轉換實驗協助時觸發。

## 使用步驟

1. 定義假設（格式：「若我們改變 X，則指標 Y 將因為理由 Z 而提升／降低」）
2. 選擇主要指標（一個主要指標 + 1–2 個護欄指標）
3. 計算所需樣本量（基準轉換率、最小可偵測效應 MDE、95% 顯著性、80% 統計效力）
4. 設計測試版本（控制組 A 維持現狀，測試組 B 僅改變單一元素）
5. 定義測試設置（工具、流量分配、排除規則、起止日期）
6. 撰寫分析計畫（何時停止、分群分析、決策規則）

## 規則

### 必須

- 啟動前必須計算樣本量，絕不能「跑到感覺顯著為止」
- 每個測試僅設一個主要指標
- 網頁測試至少執行完整一個商業周期（最少 7 天）
- 必須在看到任何結果前記錄假設

### 禁止

- 因某個版本看起來更好就提前停止測試（偷看問題）
- 啟動後更改測試參數
- 在同一用戶群上同時執行多個未隔離的重疊測試
- 在統計顯著性低於 95% 的情況下宣告勝出，且不明確說明風險

## 範例

### 正確用法

使用者：「我想測試將 CTA 從『開始免費試用』改為『免費開始』。我們的頁面每月有 500 次註冊。」

回應：撰寫假設，計算在 95% 顯著性、80% 效力、偵測 10% 提升的情況下每個版本需約 2,400 名訪客，建議最少 14 天執行期，設定註冊率為主要指標，跳出率為護欄指標。

### 錯誤用法

使用者：「讓我們測試兩個首頁標題，看看一週後哪個勝出。」

回應：未計算樣本量即同意，未設定主要指標，計劃每天查看並在「明顯勝出時」停止。
