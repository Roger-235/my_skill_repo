# competitor-alternatives

## 用途

建立高轉換率的競爭對手替代方案頁面和比較內容，捕獲來自積極評估競爭產品的用戶的高意圖底部漏斗搜尋流量。

## 觸發方式

當使用者想要建立競爭對手比較內容、為「[競爭對手]替代方案」關鍵字排名、建立比較登陸頁，或捕獲研究競爭對手的用戶流量時觸發。

## 使用步驟

1. 識別高意圖查詢（「[競爭對手]替代方案」、「[競爭對手] vs [你的產品]」、「最佳[類別]軟體」）並研究月搜尋量
2. 研究競爭對手（定價、主要功能、已知限制、G2/Capterra 評論，尤其是負評=你的定位機會）
3. 識別你的真實優勢（選擇 3–5 個你在特定使用場景或分群上客觀上更好的領域）
4. 選擇定位角度（不要定位為全面替代——定位為你勝出的特定受眾或使用場景）
5. 撰寫頁面結構（H1、[競爭對手]的優點章節、用戶尋找替代方案的原因、你如何解決這些問題、對比表、客戶推薦詞、CTA）
6. 確保所有比較聲明的準確性和可驗證性
7. 加入 FAQ 以捕獲長尾競爭對手相關查詢

## 規則

### 必須

- 所有比較中的事實必須準確——虛假聲明會帶來法律風險並損害可信度
- 承認競爭對手的優點——片面比較不可信
- 針對特定使用場景或受眾定位，而非全面更好
- 包含已切換用戶的真實客戶推薦詞

### 禁止

- 提出無法用公開數據或自身產品測試驗證的比較聲明
- 未經法律許可使用競爭對手的標誌
- 以「更便宜」作為主要差異化因素——以價值而非價格競爭
- 撰寫明顯是銷售宣傳而非真實指引的比較頁面

## 範例

### 正確用法

H1：「遠端優先團隊最佳的 Asana 替代方案」——頁面涵蓋 Asana 的優點，然後具體解決來自 Asana G2 評論的 3 個常見抱怨，展示你的產品如何解決每個問題，附功能對比表和 3 個前 Asana 用戶的推薦詞。

### 錯誤用法

「[競爭對手]很糟糕，你永遠不應該使用它。我們的產品在各方面都更好。今天就切換，享 50% 折扣。」
