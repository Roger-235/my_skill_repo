# content-creator

## 用途

作為內容創作的路由技能，根據使用者的內容類型、渠道與目標，將請求導向正確的專門內容技能，確保每種內容需求都由最適合的技能處理。

## 觸發方式

當使用者提出廣泛的內容創作需求，尚未確定具體內容類型或渠道時觸發；或當使用者詢問「我該用哪個技能來處理內容？」時觸發。

## 使用步驟

1. 詢問使用者：「你需要什麼類型的內容，以及它的目標是什麼？」
2. 從回覆中識別內容類型。
3. 對照路由表進行比對。
4. 觸發或推薦適合的技能。
5. 若請求跨越多種內容類型（如包含廣告、電子郵件和社群媒體的完整行銷活動），建議依序串接多個技能。

## 路由表

| 請求類型 | 使用的技能 | 使用時機 |
|---------|----------|---------|
| 部落格文章 / 文章 / 長篇 | `content-production` | 撰寫或起草完整文章 |
| 內容計畫 / 編輯日曆 | `content-strategy` | 規劃要創作的內容 |
| 讓 AI 內容聽起來更自然 | `content-humanizer` | 潤色 AI 生成的草稿 |
| 語法、清晰度、風格編輯 | `copy-editing` | 編輯現有草稿 |
| 銷售文案、落地頁、廣告 | `copywriting` | 說服性轉換導向文案 |
| 社群媒體貼文 | `social-content` | 社群渠道內容 |
| 電子郵件序列 / 電子報 | `email-sequence` | 電子郵件行銷內容 |
| 冷外展電子郵件 | `cold-email` | 開發客戶的電子郵件文案 |
| 廣告創意文案 | `ad-creative` | 付費廣告文案 |

## 規則

### 必須

- 始終路由至特定技能，而非嘗試通用地處理所有內容類型
- 若內容類型不明確，提問一個釐清問題
- 當專案需要多種內容類型時，建議技能串接方案

### 禁止

- 在此路由技能內嘗試執行深度內容生產工作
- 路由至不相關的技能
- 列出所有選項而不給出明確推薦

## 範例

### 正確用法

使用者：「我需要為產品上市撰寫一些東西。」→ 詢問：「這是為網站落地頁、電子郵件清單、社群媒體還是廣告？」→ 依回覆路由至 `copywriting`、`email-sequence`、`social-content` 或 `ad-creative`。

### 錯誤用法

未經路由直接在 content-creator 技能中嘗試撰寫完整部落格文章，而不觸發 `content-production`。
