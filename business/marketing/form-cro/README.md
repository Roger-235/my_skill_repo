# form-cro

## 用途

稽核並重寫網頁表單（潛客生成、註冊、結帳、聯絡表單），透過減少摩擦、漸進式揭露與改善文案和欄位設計，以提升完成率。

## 觸發方式

當使用者詢問「優化這個表單」、「提升表單轉換率」、「為何表單放棄率高」、「減少表單摩擦」，或分享表單進行 CRO 審查時觸發。

## 使用步驟

1. 稽核表單欄位（計算欄位總數、識別「可選填」vs「必填」欄位、標記可在轉換後才收集的欄位）
2. 評估欄位類型與標籤（標籤位置、欄位類型適當性、錯誤訊息、自動填入）
3. 評估 CTA（文案是否具體且以動作為導向、按鈕視覺突出度、是否只有一個主要 CTA）
4. 審查信任信號（電子郵件欄位旁的隱私說明、結帳頁的安全徽章、社會認同）
5. 應用漸進式揭露（長表單分 2–3 步驟，以最易填寫的欄位為先）
6. 撰寫優化版本（修訂欄位清單、標籤、CTA 文案與信任元素，並標注每個改變的理由）

## 規則

### 必須

- 每次稽核至少移除一個欄位，除非表單已是最精簡
- 標籤放在欄位上方，絕不僅作為佔位符文字
- 提供具體的 CTA 文案重寫，而非僅說「讓它更好」
- 在電子郵件欄位附近加入隱私微文案

### 禁止

- 在首次接觸潛客生成表單上加入 CAPTCHA，而不承認其對轉換的影響
- 除非業務有明確致電需求，否則不建議收集電話號碼
- 保留只顯示「輸入無效」的錯誤訊息
- 設計有兩個競爭性主要 CTA 的表單

## 範例

### 正確用法

使用者：「我們的註冊表單有 8 個欄位，包括名、姓、電子郵件、密碼、公司、團隊規模、職稱和電話。轉換率為 12%。」

回應：建議移除電話、公司、團隊規模和職稱（啟用後收集），合併名/姓或整個移除，精簡為電子郵件+密碼。預估轉換率提升至 25–35%，提供含信任微文案的修訂版佈局。

### 錯誤用法

使用者：「我們的註冊表單轉換率是 12%。」

回應：「把按鈕做大一點，改成綠色。」—— 無稽核、無欄位分析、無理由依據。
