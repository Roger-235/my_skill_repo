# marketing-context

## 用途

建立一份可重複使用的行銷脈絡文件，涵蓋產品定位、理想客戶輪廓（ICP）、競爭格局、通路策略與訊息架構，作為所有後續行銷工作的共同基礎。

## 觸發方式

當使用者說「設定行銷脈絡」、「定義我們的行銷策略」、「建立產品脈絡」，或在缺乏既有脈絡文件的情況下開始行銷工作時觸發。

## 使用步驟

1. 收集產品基本資訊（名稱、一句話描述、定價模式、發展階段）
2. 定義理想客戶輪廓（職稱、公司規模、產業、痛點與目標）
3. 闡述產品定位（競爭類別、差異化優勢、獨特價值主張、語調）
4. 繪製競爭格局（前 3–5 名競爭對手、優劣勢分析、常見異議）
5. 識別通路與漏斗（主要獲客通路、轉換漏斗階段、追蹤指標）
6. 輸出結構化的脈絡文件，供後續行銷技能參考使用

## 規則

### 必須

- 若缺少關鍵資訊（產品名稱、ICP），必須先提問再撰寫
- 使用具體、明確的語言，避免「創新解決方案」等空泛描述
- 以條列式呈現，保持可掃描性
- 所有假設內容需標記 `[ASSUMED]`

### 禁止

- 未經使用者確認即捏造競爭對手名稱或定價
- 輸出充滿待填寫佔位符的通用範本
- 混淆 ICP 與買家角色（Buyer Persona）
- 輸出文件超過 400 行

## 範例

### 正確用法

使用者：「我們做一個給自由接案者用的專案管理工具。競爭對手有 Notion 和 Asana。」

回應：提出 3–4 個針對性問題（定價、主要痛點、差異化），再產出包含真實定位語言的精簡脈絡文件。

### 錯誤用法

使用者：「我們做一個給自由接案者用的專案管理工具。」

回應：不問任何問題，直接輸出含有虛構 ICP 細節、假定價方案與佔位符競爭分析的完整文件。
