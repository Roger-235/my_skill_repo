# marketing-ops

## 用途

稽核並設計行銷營運基礎架構，涵蓋工具堆疊、歸因模型、自動化工作流程、數據流向與報告機制，以實現一致、可量化且可擴展的行銷執行。

## 觸發方式

當使用者請求「稽核行銷營運」、「建立行銷技術堆疊」、「改善行銷自動化」、「修復歸因問題」、「建立行銷儀表板」，或需要行銷系統與基礎架構協助時觸發。

## 使用步驟

1. 稽核現有工具堆疊（CRM、電子郵件、分析、廣告、SEO、社群等，識別缺口與重複）
2. 繪製數據流向（潛客數據來源、進入 CRM 路徑、資料豐富化與評分、數據斷點）
3. 設計或修復歸因模型（首次接觸、最後接觸或多點接觸，UTM 管理規範）
4. 定義自動化工作流程（潛客培育序列、觸發式郵件、內部警示）
5. 建立報告框架（週度運營指標、月度策略指標、儀表板工具建議）
6. 產出改善路線圖（P0 緊急、P1 重要、P2 優化，含工作量與預期影響）

## 規則

### 必須

- 依團隊規模與技術成熟度匹配工具建議
- 優先修復數據完整性問題，再考慮新增工具
- 在歸因章節中包含 UTM 命名規範指引
- 標記任何 GDPR / 隱私合規注意事項

### 禁止

- 向 10 人以下團隊推薦企業級工具（Marketo、Salesforce Marketing Cloud）而不附加說明
- 忽略現有工具，全面重建，除非使用者明確要求
- 提供缺乏工作量估算的路線圖
- 不明確推薦具體歸因模型

## 範例

### 正確用法

使用者：「我們使用 HubSpot 和 GA4。歸因出了問題——UTM 不一致，無法判斷哪個通路帶來了註冊。」

回應：稽核 HubSpot-GA4 整合，識別 UTM 不一致為根本原因，提供 UTM 命名規範範本，推薦適合其階段的首次接觸模型，並列出接下來要建立的 3 個自動化工作流程。

### 錯誤用法

使用者：「我們的行銷報告一團糟。」

回應：向一個沒有數據工程師的 3 人行銷團隊，建議導入包含 Segment、dbt 和 Looker 的完整數據倉儲。
