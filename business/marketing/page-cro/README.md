# page-cro

## 用途

對登陸頁與行銷頁面進行系統性 CRO 稽核，產出優先排序的改善清單，包含修訂文案與版面配置建議，以提高轉換率。

## 觸發方式

當使用者詢問「稽核登陸頁」、「改善頁面 CRO」、「審查首頁轉換」、「為何這個頁面不轉換」，或分享頁面 URL 或文案進行轉換審查時觸發。

## 使用步驟

1. 評估標題（是否以一句話傳達主要價值主張？是否具體且以結果為導向？是否與廣告/郵件訊息一致？）
2. 評估首屏區域（副標題支撐度、首屏 CTA 可見性、英雄圖片／影片、信任信號）
3. 稽核價值主張（前 3 大利益是否以客戶結果為中心？是否具差異化？ICP 是否被明確或隱含地指向？）
4. 檢視社會認同（客戶推薦詞的具體度、B2B 客戶標誌列、評分數量、位置是否鄰近異議與 CTA）
5. 確認異議處理（前 3 大異議是否在頁面上被處理？是否有 FAQ？風險逆轉？）
6. 評估 CTA（是否有單一主要 CTA？文案是否具體且以第一人稱表達？長頁面是否有足夠的 CTA 重複？）
7. 產出優先修復清單（P0 轉換殺手、P1 重大摩擦、P2 漸進改善）

## 規則

### 必須

- 列出問題前先為頁面評分（1–10），以提供校準基準
- 將文案問題與設計/版面問題分開
- 提供重寫後的標題、副標題與 CTA，而非僅說「改善這些」
- 考慮流量來源——付費流量需要更強的訊息一致性

### 禁止

- 建議移除社會認同——永遠只增不減
- 若首屏已擁擠，建議加入更多首屏內容
- 忽略行動裝置——單獨標記任何行動裝置特定問題
- 未識別至少一個可測試的改善就宣告頁面「良好」

## 範例

### 正確用法

使用者：「以下是我們的 SaaS 登陸頁文案。我們每月從 Google 廣告獲得 500 名訪客，但只有 1.8% 的人註冊。」

回應：識別標題以功能為中心（「AI 驅動的專案工具」）而非以結果為中心，改寫為「以 2 倍速度交付專案——不再開進度確認會議」，標記缺少 B2B 社會認同的標誌列，指出 CTA（「瞭解更多」）缺乏承諾感，改寫為「立即開始——無需信用卡」。頁面評分 4/10，包含清晰的 P0/P1 分類。

### 錯誤用法

使用者：「審查我的登陸頁 CRO。」

回應：「你的標題可以更好。加更多社會認同。讓 CTA 更突出。」—— 無具體內容、無重寫、無優先排序。
