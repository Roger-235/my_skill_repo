# prompt-engineer-toolkit

## 用途

為行銷人員提供結構化、可重複使用的提示工程框架，從 AI 工具中持續產出高品質的文案、策略、內容和分析成果。

## 觸發方式

當使用者想要為行銷工作改善 AI 提示、建立可重複使用的提示庫、從 LLM 獲得更好的行銷輸出，或學習行銷情境的提示工程時觸發。

## 使用步驟

1. 應用核心提示解剖結構：角色 + 情境 + 任務 + 格式 + 限制（不要求什麼）
2. 建立行銷情境區塊（建立包含品牌語調、ICP、產品一句話描述的可重複使用情境片段，前置於所有行銷提示中）
3. 使用少量示例（在提示中包含 1–2 個期望輸出品質的範例）
4. 指定你不想要什麼（「避免企業術語」、「不要以 I 開頭」、「不用驚嘆號」）
5. 複雜任務使用提示鏈（簡報→大綱→草稿→審查——不要在一個提示中要求一切）
6. 建立提示庫（為反覆出現的行銷任務建立並儲存可重複使用的範本）
7. 以批評迭代（第一次輸出後，提示「這個草稿最弱的 3 個元素是什麼？如何改善？」）

## 規則

### 必須

- 在所有行銷提示中包含關於目標受眾的情境——通用提示產出通用輸出
- 明確指定輸出格式（長度、結構、語調）
- 使用負面限制來防止常見的 AI 模式失敗
- 儲存並迭代產出良好結果的提示

### 禁止

- 對複雜行銷任務使用一行提示
- 不經審查/改進步驟就接受第一稿 AI 輸出
- 建立超出必要長度的提示——精準勝過長度
- 在公開 AI 工具中分享含有機密業務資訊的提示

## 範例

### 正確用法

「以資深 B2B SaaS 撰稿人身份。情境：[產品]是遠端工程團隊的專案管理工具。ICP：50–200 人公司中對狀態會議感到沮喪的工程經理。語調：直接、務實、無術語。任務：撰寫一篇關於異步溝通如何減少會議疲勞的 LinkedIn 貼文。格式：200–250 字，3–4 段短落，無標籤。避免：企業流行語、以『I』開頭、驚嘆號。」

### 錯誤用法

「幫我寫一篇關於專案管理的行銷 LinkedIn 貼文。」
