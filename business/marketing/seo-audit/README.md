# seo-audit

## 用途

對網站進行全面的 SEO 健康檢查，涵蓋技術 SEO、頁面 SEO、外部連結與內容品質，並提供優先排序的改善建議，協助提升自然搜尋流量。

## 觸發方式

當使用者需要稽核網站 SEO 狀況、診斷排名下滑原因、產出 SEO 報告，或識別影響自然搜尋表現的問題時觸發。

## 使用步驟

1. 提供目標網站 URL 及稽核範圍（全站或特定區塊）。
2. 提供 Google Search Console 匯出資料或爬蟲報告（可選，可大幅提升稽核深度）。
3. 說明已知問題或業務目標（例如：流量下滑、特定頁面排名差）。
4. 技能將依序稽核技術 SEO、頁面 SEO、外部連結及內容品質。
5. 所有問題依「嚴重 / 高 / 中 / 低」優先級分類，並附上具體修復建議。
6. 輸出執行摘要、問題列表及優先行動計畫。

## 規則

### 必須

- 依實際流量影響程度排列問題優先順序
- 每個問題需引用具體頁面或數據佐證
- 區分速效改善（一週內可完成）與長期策略項目
- 使用非技術人員也能理解的語言撰寫報告
- 每個稽核類別提供健康評分或摘要評級

### 禁止

- 建議任何黑帽 SEO 手法（隱藏文字、PBN、關鍵字堆砌）
- 在無資料佐證時捏造爬蟲數據（應說明假設前提）
- 在任何稽核中忽略行動裝置效能
- 提供沒有解釋原因的建議

## 範例

### 正確用法

「首頁的 title tag 長度為 78 個字元，在搜尋結果頁會被截斷。建議修正：縮短至 60 字元以內，並將主要關鍵字「專案管理軟體」前置。影響程度：高。執行難度：低。」

### 錯誤用法

「你的 SEO 很差。你應該加更多關鍵字，並獲得更多外部連結。」
