# competitive-teardown

## 用途

執行結構化的競品分析與產品拆解，識別競爭對手的優勢、劣勢、市場定位、定價策略與戰略空白，提供可行的競爭情報。

## 觸發方式

當使用者需要以下協助時觸發：
- 進行競品分析或市場研究
- 了解競爭對手的產品功能或定位
- 繪製市場定位圖
- 識別競爭差距與機會

## 使用步驟

1. 確認要分析的競爭對手與市場區隔
2. 為每個競爭對手建立公司概況（規模、融資、ICP、價值主張）
3. 分析產品功能、差異化特點與技術平台
4. 評估定位與訊息傳遞策略
5. 研究定價模式與策略
6. 分析通路與上市策略（GTM）
7. 對每個競爭對手進行 SWOT 分析
8. 識別市場空白與機會
9. 提出差異化策略建議

## 規則

### 必須
- 基於公開可取得的資訊（官網、定價頁、評論、職缺、新聞）
- 區分事實與推論（推論標記為「訊號」或「可能」）
- 從每個競爭對手的弱點中提出至少一個機會
- 即使只是估算，也要涵蓋定價資訊

### 禁止
- 將推論呈現為確認的事實
- 分析僅限於功能——必須包含定位與 GTM
- 忽視間接或新興競爭對手
- 在沒有策略理由的情況下建議複製競爭對手功能

## 範例

### 正確用法
```
競爭對手：Notion
ICP：知識工作者、中小企業、新創公司
價值主張：取代文件 + 知識庫 + 專案工具的一體化工作空間
定價：免費方案 → Plus $8/座位/月 → Business $15/座位/月
優勢：極強的靈活性、強大的 PLG 增長模式、豐富的模板生態系
劣勢：大規模效能問題、原生任務管理薄弱、新手上手複雜
您的機會：企業合規功能（SSO、審計日誌）有限——若目標是受監管行業則是絕佳機會
```

### 錯誤用法
```
Notion 是個熱門工具，功能很多。你應該像 Notion 一樣加入深色模式。
```
