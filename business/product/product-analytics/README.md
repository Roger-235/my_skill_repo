# product-analytics

## 用途

建立完整的產品分析策略，包含 KPI 框架、漏斗分析、留存同期群分析及事件追蹤計畫，將使用者行為與業務成果連結。

## 觸發方式

當使用者需要以下協助時觸發：
- 定義產品核心指標（North Star Metric）
- 設計事件追蹤計畫
- 分析轉換漏斗或留存率
- 建立產品指標框架或儀表板規格

## 使用步驟

1. 定義北極星指標（最能代表向使用者傳遞核心價值的單一指標）
2. 建立指標樹（使用 AARRR 框架分解北極星指標）
3. 設計轉換漏斗（從獲取到啟用到核心行動，附預期轉換率基準）
4. 定義關鍵事件（事件名稱、屬性、觸發條件）
5. 建立同期群分析框架（同期群類型、留存窗口、流失閾值）
6. 識別領先指標與滯後指標
7. 設定指標目標（基準與改善目標）
8. 指定分析維度（方案、地區、同期群、渠道）
9. 設計儀表板結構（主管視圖、產品視圖、留存視圖）

## 規則

### 必須
- 每個產品只定義一個北極星指標
- 區分領先指標（預測性）與滯後指標（結果性）
- 包含啟用指標——這是許多團隊忽略的關鍵階段
- 每個指標都要對應到一個具體的產品決策

### 禁止
- 追蹤沒有定義屬性的事件——僅有事件名稱是不夠的
- 以虛榮指標（頁面瀏覽量、應用開啟次數）作為主要成功指標
- 每個 AARRR 階段定義超過 5 個輸入指標
- 在同一漏斗步驟中混用使用者層級與 Session 層級指標

## 範例

### 正確用法
```
北極星指標：每週活躍團隊數（過去 7 天內有 ≥3 名成員完成 ≥1 次協作任務的團隊）

啟用漏斗：
1. 註冊 → 2. 建立工作區（70%）→ 3. 邀請 1 名成員（45%）→ 4. 共同完成首個任務（28%）
啟用 = 在註冊後 7 天內到達步驟 4

關鍵事件：「task_completed」
屬性：task_id、assignee_count、workspace_id、time_to_complete_ms、is_first_task
```

### 錯誤用法
```
追蹤頁面瀏覽量和每日活躍用戶。如果 DAU 上升，代表產品表現良好。
```
