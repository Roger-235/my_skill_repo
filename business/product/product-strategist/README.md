# product-strategist

## 用途

制定產品策略，包含願景、策略性投注、市場定位、平台思維、自建/購買/合作分析，以及多年期路線圖方向，建立持久的競爭優勢。

## 觸發方式

當使用者需要以下協助時觸發：
- 定義或優化產品策略
- 建立產品願景陳述
- 評估策略選項（自建、購買或合作）
- 建立多年期路線圖敘事

## 使用步驟

1. 表達產品願景（以客戶成果為導向，展望 3–5 年的未來）
2. 評估策略格局（波特五力或 JTBD 市場規模；識別主要成長向量）
3. 定義策略性投注（3–5 個高層級計畫）
4. 評估自建/購買/合作（針對每個能力缺口提出建議）
5. 平台 vs. 產品分析（評估平台層是否能建立護城河和網路效應）
6. 排列路線圖順序（在「現在/下一步/未來」視野上安排策略性投注）
7. 識別護城河（每個投注建立的持久競爭優勢）
8. 對策略進行壓力測試（預先事後分析：哪些假設必須成立？）
9. 撰寫策略敘事（一頁主管對齊摘要）

## 規則

### 必須
- 以客戶成果為根基建立願景，而非技術
- 明確列出策略性投注，盡可能相互排斥
- 解決競爭差異化——沒有差異化的策略只是計畫
- 每個策略性投注至少包含一個護城河分析

### 禁止
- 將策略與功能路線圖混淆
- 將所有策略性投注標為「高優先」——必須強制排序
- 跳過預先事後分析——未經測試的假設是策略失敗的主因
- 撰寫適用於市場上任何公司的願景陳述

## 範例

### 正確用法
```
願景：到 2028 年，每個中型市場財務團隊都能在一天內完成月末結帳——不需要試算表——因為我們的平台已學習他們的工作流程，並自動化 80% 的重複性工作。

策略性投注 1：掌控資料層（與 50+ 個來源系統整合）→ 護城河：切換成本、資料網路效應
策略性投注 2：AI 原生工作流程自動化 → 護城河：來自使用者修正的專有訓練資料
```

### 錯誤用法
```
願景：成為世界上最好的財務軟體公司。
策略：建立更多功能、改善 UI、擴展至企業市場。
```
