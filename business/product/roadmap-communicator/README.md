# roadmap-communicator

## 用途

將內部產品路線圖轉化為適合不同受眾的溝通材料，包括主管簡報、客戶面向路線圖、工程團隊視圖與利害關係人更新敘事，建立對齊、管理期望並激發信心。

## 觸發方式

當使用者需要以下協助時觸發：
- 向主管、客戶、銷售或工程師呈現產品路線圖
- 撰寫路線圖更新郵件或報告
- 建立解釋排序「原因」的路線圖敘事
- 準備路線圖溝通材料

## 使用步驟

1. 識別受眾及其關注點（主管關注成果；客戶關注痛點解決；工程師關注清晰度；銷售關注可協助成交的時程）
2. 決定承諾層級（主題式、現在/下一步/未來、或日期里程碑）
3. 撰寫策略敘事（「為何是現在」的故事）
4. 根據受眾選擇適當的路線圖視圖格式
5. 起草溝通內容（背景 → 主要主題 → 現在/下一步/未來 → 不做的事及原因 → 如何提供意見回饋）
6. 準備常見問答與異議處理
7. 定義意見回饋渠道

## 規則

### 必須
- 為主管和客戶使用成果導向語言（「更快的結帳速度」而非「新的匯出功能」）
- 明確說明路線圖上沒有的內容及原因——這能防止重複性要求
- 承諾語言必須符合實際信心水準
- 包含主要排序決策的「為何這樣排序」理由

### 禁止
- 向所有受眾展示相同的功能層級路線圖
- 在信心不足的情況下在客戶面向路線圖中承諾具體日期
- 在沒有策略「原因」敘事的情況下呈現路線圖
- 在客戶或主管溝通中使用內部代號或術語

## 範例

### 正確用法
```
[客戶面向主題卡]
主題：加速月末結帳
我們正在解決的問題：對帳讓您的團隊每個月需要花費 3 天以上。
狀態：進行中——預計 Q3 推出
包含內容：從您的會計工具自動同步、一鍵差異報告、符合審計要求的 PDF 匯出
```

### 錯誤用法
```
Q3：上線鳳凰計畫（後端重構）、JIRA-1234 匯出模組、設計系統 v2，以及銷售一直要求的那個東西。
```
