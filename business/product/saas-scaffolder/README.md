# saas-scaffolder

## 用途

為新 SaaS 產品快速搭建核心結構框架，包含定價方案、上手流程、核心功能集、商業模式畫布與上市基礎，協助創辦人和產品經理在開始建造之前對齊業務模式。

## 觸發方式

當使用者需要以下協助時觸發：
- 設計新 SaaS 產品的整體架構
- 定義定價方案與計費策略
- 規劃 SaaS 商業模式
- 設計產品上手流程與核心功能集

## 使用步驟

1. 定義理想客戶輪廓（ICP）：主要買家、最終使用者、公司規模、行業、核心痛點
2. 識別價值指標（隨客戶價值增加而擴展的計費單位）
3. 設計定價方案（通常三個：自助服務入門版、團隊成長版、企業自訂版）
4. 設計免費增值/試用策略（免費方案 vs. 免費試用；時長；轉換觸發點）
5. 繪製上手流程（從註冊到「啊哈時刻」的步驟，目標：10 分鐘內獲得首次價值）
6. 定義核心功能集（按方案分級的 MVP 功能；識別驅動升級的殺手功能）
7. 建立商業模式畫布（九個區塊）
8. 設定財務基準（CAC、LTV、LTV:CAC、回收期、流失率目標）
9. 識別整合生態系（消除採用阻力的前 5 大整合）

## 規則

### 必須
- 以隨客戶成功而擴展的價值指標定價，而非任意的座位數量
- 包含「升級觸發點」——驅動方案升級的功能或限制
- 設計上手流程在要求付款資訊之前先傳遞價值
- 明確定義「啊哈時刻」

### 禁止
- 建立超過 4 個定價方案（複雜性會影響轉換）
- 設計超過 7 個步驟才能獲得首次價值的上手流程
- 使用「請聯絡我們了解定價」而不說明進入企業方案的條件
- 忽視整合生態系——SaaS 產品的成敗取決於生態系契合度

## 範例

### 正確用法
```
價值指標：活躍追蹤專案數
定價方案：
- 入門版：免費 | 3 個專案 | 1 位使用者 | 核心追蹤功能
- 成長版：$29/月 | 20 個專案 | 5 位使用者 | 報表 + 整合 [升級觸發：專案數限制]
- 商業版：$99/月 | 無限 | 25 位使用者 | API + 自訂欄位 + 優先支援
- 企業版：自訂 | 無限 | SSO + SLA + 專屬客戶成功經理
```

### 錯誤用法
```
定價：基本版 $10/月，專業版 $50/月。功能：基本版有的東西，加上專業版的更多內容。企業版：聯絡我們。
```
