# ui-design-system

## 用途

設計 UI 設計系統規格，包含元件庫、設計令牌（Design Tokens）、間距比例、字型系統、色彩調色盤及無障礙標準，確保產品介面的視覺一致性。

## 觸發方式

當使用者需要以下協助時觸發：
- 建立或記錄設計系統
- 定義設計令牌（顏色、字型、間距）
- 規格化元件庫
- 建立無障礙設計標準
- 審計現有介面的一致性

## 使用步驟

1. 定義設計原則（3–5 條描述介面應有的感受與行為的指導原則）
2. 建立色彩系統（主色、輔助色、語義色、中性灰；定義亮色/深色模式令牌）
3. 定義字型比例（字體家族、模組化字型比例、行高、字重、使用規則）
4. 建立間距比例（基準單位 4px 或 8px；具名步驟 xs/sm/md/lg/xl/2xl）
5. 定義設計令牌（顏色、字型、間距、圓角、陰影、z-index 為具名變數）
6. 規格化核心元件（按鈕、輸入框、下拉選單、核取方塊、模態框、卡片、表格、導航、徽章、Toast）
7. 無障礙標準（WCAG AA 對比度、焦點狀態、鍵盤導航、ARIA 標籤要求）
8. 動態與動畫（時長比例、緩動函數、減少動畫處理）
9. 文件標準（Storybook 結構、元件命名規範、正確/錯誤用法範例）

## 規則

### 必須
- 以語義名稱定義令牌（--color-primary-action）而非原始值（--blue-500）
- 符合 WCAG AA 對比度最低要求（一般文字 4.5:1，大文字 3:1）
- 所有互動元件包含停用、懸停、焦點和錯誤狀態
- 為所有動畫指定減少動畫替代方案

### 禁止
- 在元件規格中硬編碼顏色值——必須使用令牌
- 建立沒有說明使用時機的元件變體
- 省略互動元件的鍵盤導航要求
- 定義色彩調色盤時沒有語義映射

## 範例

### 正確用法
```
色彩令牌：--color-feedback-error
值：#D93025（亮色）/ #FF6B6B（深色）
白底對比度：5.2:1 ✓ WCAG AA
用途：表單驗證錯誤、破壞性操作警告
正確用法：用於錯誤訊息和錯誤狀態邊框
錯誤用法：請勿用於一般警告——請改用 --color-feedback-warning
```

### 錯誤用法
```
錯誤用紅色，按鈕用藍色。確保東西看起來漂亮且一致。
```
