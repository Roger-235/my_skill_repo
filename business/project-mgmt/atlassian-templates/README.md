# atlassian-templates

## 用途

為 Jira 問題類型和 Confluence 頁面建立即用型範本，包含衝刺範本、回顧頁面、事故報告、專案啟動文件等，標準化團隊工作流程並提升資訊品質。

## 觸發方式

當使用者需要以下協助時觸發：
- 建立 Jira 問題範本（Bug、Story、Epic）
- 建立 Confluence 頁面範本
- 設計衝刺儀式範本（規劃、回顧、評審）
- 建立事故報告或專案啟動文件範本

## 使用步驟

1. 確認範本類型與使用案例
2. 定義必填欄位（範本必須捕捉的強制資訊）
3. 定義選填/情境欄位
4. 起草範本結構（清楚的標題、佔位符文字、行內指引）
5. 加入填寫說明（每個欄位的簡短行內提示）
6. 指定 Jira 欄位配置（問題類型、自訂欄位、預設值、必填欄位）
7. 定義標籤與元件（過濾和報表的標準分類）
8. 以實際案例驗證（確認範本涵蓋所有所需資訊）

## 規則

### 必須
- 在每個章節包含行內指引文字（用斜體或括號說明填寫內容）
- 讓必填欄位在視覺上與選填欄位有所區別
- 保持範本簡潔——過長的範本會被跳過
- 所有 Confluence 頁面範本包含「最後更新 / 負責人」欄位

### 禁止
- 為應該是結構化欄位的資料建立需要自由格式敘述的範本
- 在共用範本中使用未加解釋的內部術語
- 將每個欄位都設為必填——範本疲勞會降低採用率
- 在衝刺和 Story 範本中省略「完成定義」章節

## 範例

### 正確用法
```
[Jira Bug 報告範本]
## 摘要
[一句話：什麼壞了、在哪裡、對誰影響]

## 重現步驟
1. [第一步]
2. [第二步]
3. [繼續...]

## 預期行為
[應該發生什麼]

## 實際行為
[實際發生什麼——逐字包含錯誤訊息]

## 環境
- 版本：[應用版本]
- 瀏覽器/作業系統：[如適用]
- 使用者角色：[如與權限相關]

## 嚴重性
[ ] P1 – 生產環境中斷  [ ] P2 – 主要功能損壞  [ ] P3 – 次要問題  [ ] P4 – 外觀問題
```

### 錯誤用法
```
Bug 範本：描述 Bug。加入步驟。標記嚴重性。
```
