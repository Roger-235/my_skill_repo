# Financial Modeling — 財務建模

結構化財務建模框架，涵蓋收入與成本假設、損益表預測、現金流分析、單位經濟效益，以及情境分析；選擇性包含 DCF 估值。

## 功能

- 建立收入模型（由下而上或由上而下，Year 1 月度、Year 2–3 季度）
- 分離 COGS、OpEx（G&A / S&M / R&D）與 CapEx，識別固定與變動成本
- 構建完整損益表：Revenue → 毛利 → EBITDA → EBIT → 淨利
- 建立三表現金流（營運、投資、融資）並計算現金跑道
- 計算 SaaS / 訂閱模型的單位經濟效益（CAC、LTV、LTV/CAC、回收期）
- 執行 DCF 估值（自由現金流折現、終值、NPV）
- 強制執行情境分析：基礎 / 樂觀 / 悲觀三情境並排比較

## 觸發時機

**手動觸發**：
- 「財務建模」、「收入預測」、「現金流分析」、「財務預測」
- financial model、DCF、revenue projection、P&L、cash flow model、unit economics

**不觸發**：
- 純市場規模估算（不含財務預測）→ 使用 `market-research`
- 會計、記帳或報稅任務

## 使用方法

提供商業模式描述與已知或估算的財務參數。例如：

> 「幫我做一個 SaaS 新創的三年財務模型，定價 $49/月，預計每月新增 500 個用戶，月流失率 5%，初始現金 $500K。」

Claude 會依序完成：假設文件化 → 收入模型 → 成本模型 → 損益表 → 現金流 → 單位經濟效益 → 情境分析。

## 前置需求

- 商業模式描述（公司如何賺錢）
- 收入驅動因子：定價、銷售量、成長率（可為估算值）
- 成本結構：COGS、營運費用、資本支出（可為估算值）
- 建模時間軸（預設：3 年）

## 輸出格式

報告包含以下章節：
- **Key Assumptions**：所有假設與其依據來源
- **P&L Summary**：年度損益彙總表
- **Unit Economics**：CAC / LTV / LTV/CAC / 回收期
- **Cash Runway**：燃燒率與現金跑道
- **DCF Valuation**（選填）：折現率、終值、NPV
- **Scenarios**：基礎 / 樂觀 / 悲觀三情境對照
- **Assumptions & Risks**：關鍵假設的敏感性與不確定性
