# agent-designer

## 用途

設計完整的自主 AI 代理規格，涵蓋目標定義、記憶架構、工具選擇與執行循環，產出可直接實作的設計文件。

## 觸發方式

當使用者要求：
- 設計、建構或規劃 AI 代理或自主代理
- 規劃代理的工具選擇或記憶策略
- 建立以 LLM 為基礎的代理系統或 AI 助理
- 觸發關鍵詞：「design agent」、「build agent」、「agent architecture」、「autonomous agent」、「設計代理」

## 使用步驟

1. 確認代理目標與成功標準
2. 選擇代理模式（ReAct、Plan-and-Execute、Tool-Calling、Multi-Agent）
3. 設計記憶架構（上下文視窗、向量資料庫、情節記憶、語意記憶）
4. 選擇工具並說明納入理由
5. 定義執行循環（思考→行動→觀察）與終止條件
6. 規劃安全防護措施與人工介入檢查點
7. 設計評估指標
8. 輸出結構化代理設計文件

## 規則

### 必須
- 必須定義終止條件，防止無限循環
- 每個可修改狀態的工具必須設定安全防護
- 必須納入至少一個人工介入檢查點（高風險操作）
- 必須說明每項工具的納入理由
- 必須包含評估指標

### 禁止
- 禁止設計沒有錯誤恢復路徑的代理
- 禁止未說明理由直接推薦代理模式
- 禁止省略評估指標
- 禁止給予超出最低必要範圍的工具權限

## 範例

### 正確用法

```
使用者：設計一個監控 Slack 並從錯誤報告建立 Jira 票單的代理。
→ 輸出完整設計文件，包含代理類型、記憶架構、工具清單、執行循環、防護措施與評估指標。
```

### 錯誤用法

```
使用者：設計一個監控 Slack 並建立 Jira 票單的代理。
回應：代理會監控 Slack，用 AI 理解訊息，然後在需要時建立 Jira 票單。
→ 錯誤原因：無模式選擇、無記憶規格、無工具清單、無執行循環、無防護措施、無評估指標。
```
