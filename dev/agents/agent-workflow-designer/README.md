# agent-workflow-designer

## 用途

設計多步驟代理工作流程，將代理呼叫、工具調用、條件分支與平行執行編排成可生產部署的管道，並輸出步驟表、依賴圖與錯誤處理策略。

## 觸發方式

當使用者要求：
- 設計或規劃涉及多個代理步驟或工具呼叫的工作流程
- 編排 LLM 呼叫序列或代理交接
- 建立含有分支、循環或平行執行的代理管道
- 觸發關鍵詞：「agent workflow」、「multi-step agent」、「agent pipeline」、「orchestrate agents」、「工作流程設計」

## 使用步驟

1. 定義工作流程目標、輸入與輸出
2. 列舉所有離散步驟並分類（LLM 呼叫、工具呼叫、人工審查、分支）
3. 識別步驟間的依賴關係（序列 vs 可平行）
4. 設計分支邏輯，確保條件涵蓋所有情況
5. 規劃代理交接資料格式與型別
6. 規劃每個外部呼叫的錯誤處理與重試策略
7. 識別可平行化的最佳化機會
8. 輸出結構化工作流程規格與 Mermaid 依賴圖

## 規則

### 必須
- 每個步驟必須有明確的輸入與輸出 Schema
- 每個外部呼叫必須有至少一條錯誤處理路徑
- 分支條件必須涵蓋所有可能結果
- 平行步驟必須在依賴圖中明確標示
- 步驟間資料交接必須有型別定義

### 禁止
- 禁止設計有步驟但無失敗模式定義的工作流程
- 禁止步驟間隱式傳遞資料
- 禁止輸出工作流程圖而不附帶步驟表
- 禁止在存在平行化機會時省略延遲或成本影響說明

## 範例

### 正確用法

```
使用者：設計一個客服機器人工作流程，從知識庫擷取文章並生成回應。
→ 輸出完整工作流程文件，含步驟表、Mermaid 依賴圖、分支邏輯、交接 Schema 與錯誤處理。
```

### 錯誤用法

```
使用者：設計一個客服機器人工作流程。
回應：機器人會先理解問題，然後查資料，再生成答案並發送。
→ 錯誤原因：無步驟表、無 Schema、無依賴圖、無分支條件、無錯誤處理。
```
