# Auto Orchestrator — 自動多 Agent 平行執行

自動將複雜請求拆解成獨立子任務，同時派出多個 Claude agent 平行處理，最後彙整為一份統一結果。

## 功能

- 分析請求，識別可獨立執行的子任務（不互相依賴）
- 在**單一回應**中同時呼叫所有 `Task()`，實現真正平行執行
- 根據任務性質自動選擇 agent 類型（Explore / Plan / Bash / general-purpose）
- 彙整所有 agent 結果，解決矛盾，產出統一報告

## 觸發時機

- 「平行執行」、「同時做」、「多 agent 一起」、「fan out」
- 「幫我同時審查安全、效能和測試覆蓋率」
- 請求中明顯包含多個可獨立執行的部分

**不觸發**：
- 步驟 B 需要步驟 A 的輸出 → 改用 `pipeline-orchestrator`
- 簡單單步任務
- 使用者想逐步確認每個步驟

## 使用方法

直接說出你要做的事：

```
「幫我同時分析這個 repo 的安全漏洞、效能問題、和缺少測試的地方」
→ auto-orchestrator 自動派出 3 個 Explore agent 同時分析
→ 彙整三者結果輸出統一報告
```

## 關鍵設計

**為什麼要在同一回應中呼叫所有 Task()？**

Claude Code 在同一回應中的多個 Tool call 可以**同時執行**。如果分開呼叫（先等第一個回來再呼叫第二個），就變成串行執行，失去平行效益。

**上限 6 個 agent**：超過 6 個效益遞減，改用 `orchestrating-swarms` 的蜂群模式。

## 安全性

- `Task` 工具被拒絕時自動降級為串行執行並告知使用者
- 每個 agent 的 prompt 必須完整自包含，不依賴主 context
- 彙整時明確解決 agent 間矛盾，不直接拼接原始輸出
