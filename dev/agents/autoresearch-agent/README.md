# autoresearch-agent

## 用途

自主規劃並執行多輪迭代研究流程，整合多來源資訊，產出包含引用、信心評級與侷限說明的結構化研究報告。

## 觸發方式

當使用者要求：
- 深入研究或自動調查某個主題
- 蒐集、彙整或調查特定議題的資訊
- 產出研究報告或文獻摘要
- 觸發關鍵詞：「research topic」、「deep dive into」、「auto research」、「autonomous research」、「compile research on」、「研究主題」、「深入調查」

## 使用步驟

1. 將研究問題分解為 3–7 個子問題
2. 為每個子問題規劃 2–3 個針對性搜尋查詢
3. 迭代執行搜尋，評估結果相關性，發現新角度時加入查詢佇列
4. 擷取並結構化每個相關來源的發現（主張、證據、URL、日期、可信度）
5. 識別來源間的矛盾與資訊空白
6. 跨來源整合發現，依證據品質而非頻率加權
7. 為每個主要發現評定信心等級（高 / 中 / 低）
8. 輸出結構化研究報告

## 規則

### 必須
- 搜尋前必須先分解研究問題
- 每個引用來源必須評估可信度
- 必須標示來源間的矛盾，不得靜默選擇其中一方
- 必須包含侷限說明章節
- 必須為主要發現指派信心等級
- 複雜主題必須多輪迭代搜尋

### 禁止
- 禁止引用沒有 URL 和日期的來源
- 禁止將所有來源視為同等可信
- 禁止省略與發現相矛盾的證據
- 禁止捏造引用，無法找到來源時必須標記為未驗證
- 禁止以新近度替代準確性評估

## 範例

### 正確用法

```
使用者：研究 LLM 推理能力的現況。
→ 輸出完整研究報告，含研究子問題、方法論說明、按子問題結構化的發現（含證據表與信心等級）、整合摘要、信心評估、侷限說明與參考文獻。
```

### 錯誤用法

```
使用者：研究 LLM 推理能力。
回應：LLM 最近在推理方面進步了很多。Chain-of-thought 提示很有幫助。GPT-4 和 Claude 現在被認為擅長推理。
→ 錯誤原因：無子問題、無搜尋方法、無引用、無來源可信度評估、無信心等級、無矛盾說明、無侷限說明。這是無來源的意見，不是研究。
```
