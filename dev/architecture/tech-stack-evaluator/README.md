# tech-stack-evaluator

## 用途

針對技術選型決策定義加權評估標準，對候選技術逐項評分，計算加權總分，識別風險，並產出附有明確推薦理由的比較矩陣報告。

## 觸發方式

當使用者要求：
- 在兩個或多個技術、框架、資料庫或語言之間做選擇
- 評估或比較專案的技術堆疊
- 獲得應使用哪種技術的建議
- 觸發關鍵詞：「evaluate tech stack」、「choose technology」、「compare frameworks」、「select database」、「technology decision」、「stack comparison」

## 使用步驟

1. 定義決策背景（技術決策內容、專案情境、團隊限制、硬性條件）
2. 列舉所有候選技術（不足時補充 1–2 個知名替代方案）
3. 定義評估標準（5–8 項）並分配權重（合計 100%）
4. 對每個候選技術逐項評分（1–5 分）並記錄評分理由
5. 計算加權分數（分數 × 權重，加總）
6. 識別每個候選技術的採用風險
7. 提出推薦（最高加權分，說明選擇理由及次選何時更適合）
8. 輸出評估報告

## 規則

### 必須
- 評分前必須定義明確的評估標準與權重
- 各標準權重合計必須等於 100%
- 每個評分都必須提供理由
- 每個候選技術至少列出 2 個風險
- 必須說明何種情況下次選方案更合適
- 推薦必須基於評分結果，不得基於個人偏好

### 禁止
- 禁止未進行評分比較直接推薦技術
- 禁止所有標準使用相同權重（代表無優先思考）
- 禁止遺漏使用者應知道的重要替代方案
- 禁止忽略已聲明的硬性條件（授權、合規、語言）

## 範例

### 正確用法

```
使用者：我們的電商產品目錄應該使用 PostgreSQL 還是 MongoDB？3 人團隊，現有 Python 技術堆疊。
→ 輸出完整評估報告：定義 5 項加權標準、逐項評分（含理由）、計算加權總分、風險分析、明確推薦 PostgreSQL（含理由）及何時應選 MongoDB 的條件說明。
```

### 錯誤用法

```
使用者：我應該用 PostgreSQL 還是 MongoDB？
回應：PostgreSQL 適合結構化資料，MongoDB 適合靈活 Schema，視使用情境而定。
→ 錯誤原因：無標準定義、無權重、無評分、無專案情境、無風險分析、無明確推薦，「視情況而定」卻未說明什麼情況。
```
