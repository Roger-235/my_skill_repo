# mcp-server-builder

## 用途

根據能力規格建構完整的生產就緒 MCP 伺服器實作，包含帶型別的工具定義、輸入驗證、錯誤處理、安全強化，以及即用的連線說明。

## 觸發方式

當使用者提供能力規格並要求建構 MCP 伺服器、實作 MCP 工具，或從描述生成完整 MCP 伺服器時觸發。也由 `auto-mcp` 在路由能力缺口時觸發。

## 使用步驟

1. 解析能力規格——提取工具清單、外部服務、認證方法、資料格式
2. 設計工具介面——動詞片語命名、帶描述的型別輸入 Schema、輸出格式
3. 搭建伺服器結構（Python: 單檔案 / TypeScript: src/ 目錄結構）
4. 實作伺服器——含 MCP SDK 初始化、輸入驗證、環境變數憑證、TODO 存根
5. 應用安全強化——無動態輸入的 shell=True、路徑遍歷檢查、不可逆操作守護
6. 生成連線配置（claude_desktop_config.json 片段）
7. 產出帶編號的 TODO 清單

## 規則

### 必須
- 生成完整、語法正確、可執行的伺服器檔案，而非偽代碼
- 每個工具處理器包含輸入驗證
- 透過環境變數引用所有憑證，絕不硬編碼
- 為需要整合特定邏輯的地方包含 `# TODO:` 存根
- 伺服器代碼旁附上 MCP 配置 JSON 區塊
- 印出帶編號的設置清單

### 禁止
- 生成以動態輸入搭配 `shell=True` 執行任意 shell 指令的工具
- 硬編碼 API 金鑰、密碼、令牌或連線字串
- 生成沒有注釋守護條件的變更性工具（寫入、刪除、發送）
- 跳過錯誤處理——每個工具失敗時必須返回結構化錯誤回應
- 生成靜默吞噬例外的代碼

## 範例

### 正確用法

能力規格：「用只讀 SQL 查詢查詢 PostgreSQL 資料庫的工具。」

生成 `postgres-mcp.py`，含 `query_db(sql: str)` 工具，驗證 SQL 以 SELECT 開頭，使用 `DATABASE_URL` 環境變數的 psycopg2 連線，將結果返回為 JSON 格式文字，包含 `# INSTALL:` 注釋，連線失敗時返回結構化錯誤。附 MCP 配置 JSON 與 4 項設置清單。

### 錯誤用法

生成片段 `import psycopg2; conn = psycopg2.connect("postgresql://admin:password@localhost/db")`——硬編碼憑證、無 MCP 結構、無輸入驗證、無錯誤處理。
