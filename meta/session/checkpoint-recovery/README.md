# Checkpoint Recovery — 斷點恢復

儲存長任務的進度快照，在中斷、上下文重置或工作階段結束後，能精確恢復並繼續執行，不重複已完成的工作。

## 功能

- 自動偵測需要儲存檢查點的時機（剩餘步驟多、上下文快滿、即將執行不可逆操作）
- 記錄完整任務狀態：已完成步驟與成果、當前步驟狀態、剩餘步驟、關鍵決策、已建立或修改的檔案
- 將檢查點寫入 `checkpoint.md`；若無寫入權限，以內文形式輸出供使用者手動儲存
- 恢復時驗證當前檔案系統狀態與檢查點的一致性，發現差異主動回報
- 每完成一個步驟後更新檢查點，任務全部完成後刪除檢查點檔案

## 觸發時機

**手動觸發**：
- 「儲存進度」、「設定斷點」、「繼續任務」、「從哪裡繼續」
- save progress、checkpoint、resume task、where were we、recover from interruption

**自動觸發**：
- 剩餘子任務超過 5 個且上下文使用率超過 70%
- 即將執行不可逆的外部操作（寫檔、API 呼叫、資料庫變更）
- 使用者表示需要暫停

**不觸發**：
- 單次對話可完成的簡單任務
- 已完全完成的任務

## 使用方法

在長任務執行中途說明需要儲存進度，或直接開始新對話並要求恢復。例如：

> 「我需要先暫停，幫我存一下目前的進度。」

或在新對話中：

> 「繼續我們之前的重構任務，checkpoint.md 在專案根目錄。」

Claude 會讀取檢查點、驗證檔案狀態，並在確認後從正確的步驟繼續。

## 前置需求

- 進行中的任務，已有明確的步驟定義
- 對當前工作目錄有寫入權限（用於儲存 `checkpoint.md`）

## 輸出格式

檢查點檔案（`checkpoint.md`）包含：
- **Status & Goal**：任務狀態、目標、完成條件
- **Completed Steps**：每個已完成步驟及其產出
- **Current Step**：當前步驟的進行狀態
- **Remaining Steps**：剩餘步驟清單
- **Key Decisions**：已做決策及其理由
- **Created / Modified Files**：所有受影響的檔案路徑與描述
- **Resume Instructions**：精確的恢復指令與驗證清單
