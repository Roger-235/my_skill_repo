# Continuous Learning — 持續學習記憶

從當前對話中提取可重複使用的模式，儲存至持久記憶系統，讓 Claude 在未來的工作階段中主動應用這些模式。

## 功能

- 識別並抽象化可重複使用的規則或偏好（不是一次性修正，而是未來可套用的模式）
- 讀取 MEMORY.md 索引，檢查重複記憶，若有相近記憶則更新而非重複建立
- 分類記憶類型：使用者偏好 / 行為糾正（含原因）/ 專案背景 / 參考資料
- 寫入記憶檔案（含觸發條件、理由、應用方式）並更新 MEMORY.md 索引
- 向使用者確認儲存內容、類型、路徑，以及何時會觸發

## 觸發時機

**手動觸發**：
- 「記住這個模式」、「記下來」、「以後都這樣做」、「學習記憶」
- remember this pattern、save this approach、learn from this、this is how I want it done

**自動觸發**：
- 使用者糾正 Claude 的行為，且此糾正可能在未來再次適用
- 新問題以特定方式解決，且解法明顯預期重複使用

**不觸發**：
- 一次性任務的臨時細節（如「今天的會議在 3 點」）
- 不太可能重複發生的一次性 bug 修正
- CLAUDE.md 中已記錄的資訊
- 程式碼片段（應放在程式碼庫中，不是記憶系統）

## 使用方法

直接說明要記住的模式，或在 Claude 犯錯後糾正並告知未來要如何處理。例如：

> 「記住：我們的整合測試不使用 mock，要用真實的測試資料庫。」

Claude 會確認無重複記憶後，建立記憶檔案並更新索引，然後告知你何時這個記憶會被套用。

## 前置需求

- 記憶目錄的存取權：`/home/roger/.claude/projects/`（或專案專屬記憶路徑）
- 記憶目錄中存在 `MEMORY.md` 索引檔案
- 要儲存的模式必須真正可重複使用，而非僅適用於當前任務

## 安全性

- 不儲存臨時任務細節，僅儲存跨工作階段可重複使用的模式
- 不儲存對使用者個人特質或能力的負面評價
- 每次儲存前都會檢查重複，避免衝突的行為規則
- 不覆寫 CLAUDE.md 中已記錄的資訊，維持單一真實來源
