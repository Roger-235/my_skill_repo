# Pre-Output Review — 輸出前品質把關

在每次輸出任何實質內容之前，自動執行品質檢查清單，直到全部通過才交付給使用者。

## 功能

- **自動觸發**：適用於所有實質輸出 — 程式碼、解釋、計畫、設計、文件、指令、分析
- 6 大類檢查：完整性、正確性、範疇控制、格式、資安、一致性
- 任一項目失敗 → 修正 → 重新檢查 → 循環直到全部通過
- 審查過程對使用者不可見，只輸出最終通過的結果

## 檢查清單

| 類別 | 關鍵項目 |
|------|---------|
| **完整性** | 每個子問題都有回答；沒有靜默跳過的部分 |
| **正確性** | 無虛構 API／函式；語法合法；指令 flag 有效 |
| **範疇** | 沒有超出需求的功能或檔案；不沉默地多做 |
| **格式** | code block 有語言標籤；markdown 有效；長度匹配複雜度 |
| **資安** | 無硬編憑證；無 injection 漏洞；檔案操作範圍受限 |
| **一致性** | 符合對話歷史；名稱風格符合現有代碼庫；無自我矛盾 |

## 觸發時機

**觸發**：輸出程式碼、解釋、計畫、架構設計、schema、文件、分析報告、任何使用者會行動或閱讀的內容

**不觸發**：單字確認（「好」「OK」）、是/否回答、向使用者提問、複述使用者說的話

## 安全性

- 審查過程完全在內部進行，不會輸出給使用者
- 若某項目無法解決（資訊確實未知），會在輸出中明確說明限制，而非猜測填補
- 不會因為「可能有用」就加入未被要求的內容
