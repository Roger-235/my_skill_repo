# Token Optimizer — Token 用量優化

分析當前上下文視窗的 token 使用量，識別最大消耗來源，並在達到限制之前推薦或執行優化策略。

## 功能

- 估算當前對話的 token 用量（英文 1 token ≈ 4 字元、中文 1 token ≈ 1.5–2 字元）
- 分類識別最大消耗來源：大型檔案讀取、重複內容、冗長工具輸出、低密度對話歷史
- 以三色狀態評估緊急程度：Green（<50%）/ Yellow（50–75%）/ Red（>75%）
- 針對每個消耗來源推薦具體的優化動作
- 取得使用者明確同意後才執行任何壓縮或刪除操作
- 報告優化前後的 token 節省量與百分比

## 觸發時機

**手動觸發**：
- 「Token 用量」、「節省 token」、「壓縮 context」、「context 快滿了」
- token usage、optimize context、running out of context、how much context left

**自動觸發**：
- 估算上下文使用率超過 70% 時主動提醒
- 即將執行會新增大量 token 的操作前（如讀取多個大型檔案）

**不觸發**：
- 上下文使用率低於 50% 的正常對話

## 使用方法

詢問目前的 token 使用情況，或讓 Claude 在使用率偏高時主動提醒。例如：

> 「現在用了多少 token？還夠用嗎？」

Claude 會估算使用量、列出主要消耗來源，並提出具體優化建議。所有優化動作需經你確認後才會執行。

## 前置需求

- 當前對話的上下文可供分析
- 使用者在執行任何優化動作前能夠審閱並確認

## 輸出格式

報告包含以下章節：
- **Estimated usage**：估算 token 數與百分比，以及狀態（Green / Yellow / Red）
- **Top Consumers**：消耗來源列表，含各別估算與建議動作
- **Recommendation**：優先排序的優化動作清單
- **Projected savings**：執行後的預期節省量與剩餘用量
- **Approval needed**：需使用者確認的動作清單

## 安全性

- 任何壓縮或刪除操作都需要使用者明確授權
- 不在關鍵操作進行中（寫檔、生成中、工具呼叫中）執行 `/compact`
- 傾向高估而非低估 token 用量，保護使用者不在任務中途觸及限制
