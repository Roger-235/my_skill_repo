# skill-security-auditor

## 用途

審計 SKILL.md 檔案的安全漏洞，包括提示注入向量、不安全的工具使用說明、憑證暴露模式，以及可能導致 Claude 執行有害或非預期行為的指令。

## 觸發方式

當使用者要求對技能進行安全審計、檢查 SKILL.md 是否存在注入風險或不安全指令、審查技能權限，或驗證技能是否遵循安全設計原則時觸發。

## 使用步驟

1. 解析技能結構——確認必要區段存在且 frontmatter 有效
2. 審計觸發條件——可被利用的自動觸發條件、過於寬泛的觸發器
3. 審計步驟區段——掃描提示注入向量、憑證暴露、路徑遍歷風險、Shell 注入、範圍蔓延指令、不可逆行動自動化
4. 審計規則區段——驗證 Never 清單明確禁止最高風險行動
5. 審計範例區段——確認正確範例不展示不安全模式
6. 評估整體風險狀況（Critical / High / Medium / Low）
7. 產出含位置、風險等級、描述與建議修復的發現清單

## 規則

### 必須
- 為每個發現指定風險等級
- 區分理論風險與技能環境中的實際利用路徑
- 將步驟區段的提示注入向量作為最高優先級審查
- 標記可被攻擊者控制內容啟動的自動觸發條件
- 為每個發現建議具體的修改文字

### 禁止
- 將標準技能行為（如讀取使用者明確提供的檔案）標記為安全問題
- 在未提供明確修復指導的情況下批准含 Critical 評級發現的技能
- 忽略步驟與外部資料來源（檔案、網頁內容、資料庫結果）之間的互動
- 以技能的良好意圖作為安全的依據——審計指令本身

## 範例

### 正確用法

審計 `sql-query-builder` 技能，識別：（High）步驟指示 Claude 從使用者提供的描述中推斷表名而未注意生成查詢中的 SQL 注入風險；（Medium）觸發條件過於寬泛可能在非預期情境中啟動。建議在 Never 規則中加入「絕不使用使用者輸入的字串拼接構建查詢」並縮窄觸發條件。

### 錯誤用法

回應「看起來很安全！這個技能有很好的規則。」——無系統性審計、無注入向量分析、無風險評級、無具體發現。
