# skill-tester

## 用途

透過模擬真實觸發場景測試 SKILL.md 檔案，驗證遵循技能步驟所產生的輸出是否符合規格的輸出格式、滿足規則，並展示範例中的行為。

## 觸發方式

當使用者要求測試技能、模擬技能對給定輸入的回應、驗證技能是否按設計運作，或在將技能部署到專案前進行驗證時觸發。

## 使用步驟

1. 解析技能規格——提取觸發條件、前置條件、步驟、輸出格式、規則與範例
2. 生成測試場景——至少 3 種（典型觸發、邊緣觸發、非觸發、對抗性輸入）
3. 模擬技能執行——對每個場景逐步執行技能，生成輸出
4. 評估完整性——驗證模擬輸出符合輸出格式、包含必要區段、不違反 Never 規則
5. 測試前置條件處理——模擬前置條件不滿足的場景（技能應詢問而非假設）
6. 產出測試結果——每個場景標注通過/失敗，失敗時說明違反哪個步驟或規則

## 規則

### 必須
- 為觸發區段中每個不同的觸發關鍵字/條件至少測試一個場景
- 包含嘗試違反 Never 規則的對抗性場景
- 驗證技能優雅處理缺失的前置條件（詢問而非假設）
- 為每個通過場景檢查所有 Must 規則
- 以可測試的通過/失敗標準報告發現，而非定性評估

### 禁止
- 在違反任何 Must 規則的情況下將場景標記為通過
- 跳過測試 Never 規則——這是最重要的行為限制
- 只測試快樂路徑——邊緣情況和對抗性輸入是必要的
- 因步驟看起來合理就假設技能能正常運作——模擬實際輸出
- 報告「技能似乎能運作」等模糊結果，而非按場景的具體證據

## 範例

### 正確用法

測試 `pr-review-expert`，場景「使用者分享將未驗證的查詢參數拼接進 SQL 字串的 diff」：

場景：典型觸發（代碼審查請求）——通過
- 輸入正確觸發（符合「審查這個 diff」）
- 識別 SQL 注入（字串拼接 + 使用者輸入），指定 Blocking 優先級，提供 file:line 引用與參數化查詢修復
- 所有 Must 規則已滿足；無 Never 規則違反；輸出符合輸出格式

整體結果：通過

### 錯誤用法

「我閱讀了這個技能，步驟看起來合理。範例清晰。我認為它能正常工作。」——無場景模擬、無按規則的通過/失敗、無對抗性測試、無前置條件檢查。
