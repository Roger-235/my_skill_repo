# Article Writing — 技術文章撰寫

結構化引導技術文章與部落格文章的完整撰寫流程，從讀者定義到最終格式化輸出。

## 功能

- 定義目標讀者與文章核心目標
- 建立 H2/H3 大綱（3–7 個主要段落，各有明確子目的）
- 撰寫引言（吸引注意 + 背景鋪陳 + 論點聲明）
- 每個段落以論點開頭，輔以具體範例
- 編輯消除填充語句，確保每句話都有價值
- 依目標平台格式化（Markdown、程式碼區塊、字數控制）

## 觸發時機

- 「寫文章」、「寫部落格」、「撰寫技術文章」
- "write an article", "draft a blog post", "technical writing"

**不觸發**：需要批量生產多種格式內容時，使用 `content-engine`。

## 使用方法

提供主題後，skill 會先確認讀者定義與文章目標，再建立大綱，逐段撰寫，最後編輯與格式化。

**輸出格式：**
```
## Article: <標題>

**Audience:** <目標讀者>
**Goal:** <核心收穫>

---
<文章內容>
---
**Word count:** ~X
**Reading time:** ~Y min
```

## 前置需求

- 一個撰寫主題
- （選填）目標平台與預期字數
